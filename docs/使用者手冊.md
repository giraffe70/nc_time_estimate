# NC-Time-Twin 使用者手冊

## 1. 安裝與環境準備

本專案需要 Python 3.11 以上。建議先建立虛擬環境，再安裝依賴。

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

若要以 package 形式安裝：

```powershell
pip install -e .
```

若未安裝 package，也可以在專案根目錄設定 `PYTHONPATH`：

```powershell
$env:PYTHONPATH="src"
```

## 2. 專案目錄

常用目錄如下：

- `examples/`：範例 NC-Code。
- `profiles/`：機台設定 YAML。
- `output/`：自動或手動輸出的報表。
- `logs/`：估測 log。
- `docs/`：專案文件。
- `src/nc_time_twin/`：程式碼。
- `tests/`：測試案例。

## 3. 使用 CLI 估測加工時間

基本指令：

```powershell
python -m nc_time_twin estimate --nc examples/basic.nc --profile profiles/default_3axis.yaml --out output/basic.json
```

或在安裝 package 後使用 console script：

```powershell
nc-time-twin estimate --nc examples/basic.nc --profile profiles/default_3axis.yaml --out output/basic.json
```

執行後系統會：

1. 讀取指定 NC 檔案。
2. 讀取指定機台 profile。
3. 解析 NC-Code 並估測時間。
4. 依 `--out` 輸出指定格式報表。
5. 自動額外輸出一份 Excel 報表到 `output/`。
6. 自動輸出 log 到 `logs/`。

若未指定 `--out`，CLI 會將 summary JSON 印到標準輸出，同時仍會產生自動 Excel 報表與 log。

## 4. 輸出格式

可透過 `--out` 副檔名自動推論格式：

```powershell
python -m nc_time_twin estimate --nc examples/basic.nc --profile profiles/default_3axis.yaml --out output/basic.xlsx
python -m nc_time_twin estimate --nc examples/basic.nc --profile profiles/default_3axis.yaml --out output/basic.html
python -m nc_time_twin estimate --nc examples/basic.nc --profile profiles/default_3axis.yaml --out output/basic.csv
python -m nc_time_twin estimate --nc examples/basic.nc --profile profiles/default_3axis.yaml --out output/basic.json
```

也可用 `--format` 明確指定：

```powershell
python -m nc_time_twin estimate --nc examples/basic.nc --profile profiles/default_3axis.yaml --out output/report.data --format json
```

支援格式：

- `json`
- `csv`
- `xlsx`
- `excel`
- `html`

## 5. 查看摘要

加上 `--print-summary` 可在命令列列印 summary JSON：

```powershell
python -m nc_time_twin estimate --nc examples/basic.nc --profile profiles/default_3axis.yaml --out output/basic.xlsx --print-summary
```

摘要包含：

- 總時間。
- 快速移動時間。
- 切削時間。
- 圓弧時間。
- dwell 時間。
- 換刀、主軸、切削液、optional stop 等輔助時間。
- 總路徑長。
- feed 統計與 warning count。

## 6. 指定進給單位

如果 NC 檔案中的 `G21/G94 F` 值確定是 mm/min：

```powershell
python -m nc_time_twin estimate --nc examples/basic.nc --profile profiles/default_3axis.yaml --feed-unit mm_per_min --out output/basic.xlsx
```

如果 `G21/G94 F` 值代表 m/min，例如 `F6` 代表 6000 mm/min：

```powershell
python -m nc_time_twin estimate --nc examples/basic.nc --profile profiles/default_3axis.yaml --feed-unit m_per_min --out output/basic.xlsx
```

如果希望依程式內容自動判斷：

```powershell
python -m nc_time_twin estimate --nc examples/basic.nc --profile profiles/default_3axis.yaml --feed-unit auto --out output/basic.xlsx
```

預設 profile 目前使用 `feed_unit: "auto"`。

## 7. 使用嚴格進給檢查

若要更嚴格檢查異常低速、混合尺度或極端原始 F 值：

```powershell
python -m nc_time_twin estimate --nc examples/ToolPathSource.opti.nc --profile profiles/default_3axis.yaml --strict-feed --out output/opti.xlsx
```

若希望檢查到 critical feed sanity issue 時讓 CLI 回傳失敗：

```powershell
python -m nc_time_twin estimate --nc examples/ToolPathSource.opti.nc --profile profiles/default_3axis.yaml --strict-feed --fail-on-sanity-error --out output/opti.xlsx
```

這適合用於 CI 或自動化流程，避免異常 NC 程式進入後續加工流程。

## 8. 比較優化前後 NC 程式

若要比較 candidate NC 與 source NC：

```powershell
python -m nc_time_twin estimate --nc examples/ToolPathSource.opti.nc --compare-nc examples/ToolPathSource.nc --profile profiles/default_3axis.yaml --out output/compare.xlsx
```

報表會包含：

- source 與 candidate 的總時間。
- 總時間差。
- 切削時間差。
- 幾何是否一致。
- 是否判定為時間退化。
- 各進給區間的時間差。
- 最明顯的退化 blocks。

若要在時間退化時讓 CLI 回傳 exit code 1：

```powershell
python -m nc_time_twin estimate --nc examples/ToolPathSource.opti.nc --compare-nc examples/ToolPathSource.nc --profile profiles/default_3axis.yaml --fail-on-regression --out output/compare.xlsx
```

可用 `--max-regression-ratio` 設定允許退化比例。例如允許 5%：

```powershell
python -m nc_time_twin estimate --nc candidate.nc --compare-nc source.nc --profile profiles/default_3axis.yaml --max-regression-ratio 0.05 --fail-on-regression --out output/compare.xlsx
```

## 9. 正規化進給值

若 NC 程式的 G21/G94 F 值代表 m/min，但希望輸出為明確 mm/min，可使用 `normalize-feed`。

範例：將 m/min 轉成 mm/min：

```powershell
python -m nc_time_twin normalize-feed --nc examples/ToolPathSource.opti.nc --profile profiles/default_3axis.yaml --input-feed-unit m_per_min --out output/ToolPathSource.normalized.nc --print-summary
```

若輸入本來就是 mm/min，但仍希望依 profile 的 `max_cut_feed_mm_min` 截斷過高 F 值：

```powershell
python -m nc_time_twin normalize-feed --nc input.nc --profile profiles/default_3axis.yaml --input-feed-unit mm_per_min --out output/input.normalized.nc --print-summary
```

注意事項：

- `normalize-feed` 不允許覆寫原檔，`--out` 必須是不同路徑。
- 只會改寫公制 `G21` 且每分鐘進給 `G94` 狀態下的 F 字。
- `G95`、`G93`、英制 `G20` 狀態下的 F 字會跳過。
- 註解中的 F 字不會被改寫。
- 超過 `max_cut_feed_mm_min` 的值會被截斷。

## 10. 使用 GUI

啟動 GUI：

```powershell
python -m nc_time_twin.gui.main_window
```

或安裝 package 後：

```powershell
nc-time-twin-gui
```

GUI 操作流程：

1. 在 Project 分頁按 `Browse` 選擇 NC-Code 檔案。
2. 確認 Machine Profile 欄位，預設為 `profiles/default_3axis.yaml`。
3. 按 `Estimate`。
4. 等待狀態顯示 `Done: HH:MM:SS`。
5. 到 Results 分頁查看 summary。
6. 到 Blocks 分頁查看每個 block 的行號、原始內容、起終點、長度、時間、進給等資訊。
7. 到 Charts 分頁查看 XY Toolpath 與 Block Time 圖。
8. 到 Warnings 分頁查看警告。
9. 在 Project 分頁選擇 export 格式，按 `Save Report` 手動輸出報表。

GUI 每次估測後也會自動輸出：

- Excel 報表到 `output/`
- log 到 `logs/`

## 11. 報表閱讀方式

### 11.1 Summary

Summary 是整體估測摘要。重點欄位：

- `total_time_sec`：總時間秒數。
- `total_time_text`：總時間，格式為 `HH:MM:SS`。
- `rapid_time_sec`：快速移動時間。
- `cutting_time_sec`：線性與圓弧切削時間。
- `arc_time_sec`：圓弧切削時間。
- `auxiliary_time_sec`：換刀、主軸、切削液、optional stop、reference return、dwell 的合計。
- `total_length_mm`：總路徑長。
- `warning_count`：警告數。
- `feed_unit_effective`：實際用於估測的進給單位策略。
- `feed_capped_count`：被最大切削進給上限截斷的 block 數。

### 11.2 Blocks

Blocks 是逐 block 明細。重點欄位：

- `line_no`：原始 NC 行號。
- `raw`：原始 NC 行內容。
- `type`：block 類型，例如 rapid、linear、arc。
- `start` / `end`：起點與終點。
- `length_mm`：幾何長度。
- `estimated_time_sec`：該 block 估測時間。
- `feedrate`：原始 F 值。
- `feed_mode`：G93/G94/G95。
- `feed_unit`：該 block 的進給單位解讀。
- `effective_feed_mm_min`：換算後有效 mm/min。
- `feed_capped`：是否被上限截斷。
- `warnings`：該 block 的警告。

### 11.3 Diagnostics

Excel 報表中的 diagnostics sheet 會整合：

- warnings。
- feed histogram。
- top slow blocks。
- feed sanity summary。
- feed sanity issues。
- compare 結果。

### 11.4 Log

Log 檔適合快速人工審查，內容包含：

- Summary。
- Warnings。
- Feed Histogram。
- Top Slow Feed Blocks。
- Feed Sanity Summary。
- Feed Sanity Issues。
- Comparison。

## 12. 常見情境

### 12.1 想知道單一 NC 檔加工時間

```powershell
python -m nc_time_twin estimate --nc examples/O0001.NC --profile profiles/default_3axis.yaml --out output/O0001.xlsx
```

### 12.2 優化後時間反而變長

```powershell
python -m nc_time_twin estimate --nc examples/ToolPathSource.opti.nc --compare-nc examples/ToolPathSource.nc --profile profiles/default_3axis.yaml --fail-on-regression --out output/ToolPathSource_compare.xlsx
```

檢查報表中的：

- `comparison.is_regression`
- `comparison.geometry_match`
- `top_time_regression_blocks`
- `feed_sanity_issues`
- `feed_histogram`

### 12.3 懷疑 F 值單位混用

先用嚴格模式估測：

```powershell
python -m nc_time_twin estimate --nc candidate.nc --profile profiles/default_3axis.yaml --strict-feed --out output/candidate.xlsx
```

若報表建議正規化，再執行：

```powershell
python -m nc_time_twin normalize-feed --nc candidate.nc --profile profiles/default_3axis.yaml --input-feed-unit m_per_min --out output/candidate.normalized.nc --print-summary
```

然後重新估測正規化後的 NC 檔。

### 12.4 需要調整機台速度

複製預設 profile：

```powershell
Copy-Item profiles/default_3axis.yaml profiles/my_machine.yaml
```

修改：

- `axes.X.rapid_velocity_mm_min`
- `axes.Y.rapid_velocity_mm_min`
- `axes.Z.rapid_velocity_mm_min`
- `max_cut_feed_mm_min`
- `event_time.tool_change_sec`
- 其他實際機台參數

再用新 profile 執行估測：

```powershell
python -m nc_time_twin estimate --nc examples/basic.nc --profile profiles/my_machine.yaml --out output/basic_my_machine.xlsx
```

## 13. 錯誤與警告處理

常見 warning 與建議：

- `Missing feedrate F; using default_cut_feed_mm_min`：切削移動缺少 F，請補上 F 或確認預設進給是否合理。
- `G95 requires spindle speed S; using default_cut_feed_mm_min`：G95 需要主軸轉速，請在前面設定 S。
- `R arc length is approximate in Phase 1`：R 圓弧採近似，若要求精確請改用 IJK。
- `Arc radius mismatch`：IJK 圓弧起點半徑與終點半徑差距超過容差，請檢查 NC。
- `Suspicious low G94 metric feedrates`：G21/G94 下 F 值偏低，請確認是否其實是 m/min。
- `Feedrate capped by max_cut_feed_mm_min`：F 值超過 profile 上限，估測時已截斷。
- `reference return time is not estimated`：profile 中 reference return 設為 `unestimated`。

## 14. 測試

執行測試：

```powershell
pytest
```

測試涵蓋：

- 前處理與 tokenizer。
- modal continuation。
- 絕對 / 增量座標。
- G93/G94/G95 進給。
- G00 同步軸快速時間。
- IJK/R 圓弧。
- G81/G82/G83 cycle 展開。
- JSON/CSV/Excel/HTML 報表。
- CLI compare 與 sanity failure。
- normalize-feed。
