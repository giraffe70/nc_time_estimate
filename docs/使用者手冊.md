# NC-Time-Twin 使用者手冊

## 1. 安裝與環境準備

本專案需要 Python 3.11 以上。

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

若要以 package 形式安裝：

```powershell
pip install -e .
```

若未安裝 package，也可以在專案根目錄設定 `PYTHONPATH`：

```powershell
$env:PYTHONPATH="src"
```

## 2. 專案目錄

- `examples/`：範例 NC-Code。
- `profiles/`：機台設定 YAML，包含第一階段與第二階段預設 profile。
- `output/`：預設報表輸出資料夾。
- `logs/`：估測 log。
- `docs/`：專案文件。
- `src/nc_time_twin/`：程式碼。
- `tests/`：測試案例。

## 3. 使用 CLI 估測加工時間

基本估測：

```powershell
python -m nc_time_twin estimate --nc examples/basic.nc --profile profiles/default_3axis.yaml --out output/basic.json
```

安裝 package 後：

```powershell
nc-time-twin estimate --nc examples/basic.nc --profile profiles/default_3axis.yaml --out output/basic.json
```

CLI 執行後會：

1. 讀取 NC 檔案。
2. 讀取 machine profile。
3. 解析 NC-Code 並估測時間。
4. 依 `--out` 輸出指定格式報表。
5. 額外自動輸出 Excel 報表到 `output/`。
6. 自動輸出 log 到 `logs/`。

若未指定 `--out`，CLI 會將 summary JSON 印到 stdout，同時仍會產生自動 Excel 報表與 log。

## 4. 使用第二階段 Phase 2 模型

使用 Phase 2 profile：

```powershell
python -m nc_time_twin estimate --nc examples/basic.nc --profile profiles/default_phase2_3axis.yaml --out output/basic_phase2.xlsx
```

或用 CLI 覆寫既有 profile 的時間模型：

```powershell
python -m nc_time_twin estimate --nc examples/basic.nc --profile profiles/default_3axis.yaml --time-model phase2 --out output/basic_phase2.xlsx
```

Phase 2 報表會增加：

- Phase 2 summary。
- junction 診斷。
- bottleneck 診斷。
- block 層級 entry / exit / peak velocity。
- velocity chart samples。

## 5. 輸出格式

可透過 `--out` 副檔名推論格式：

```powershell
python -m nc_time_twin estimate --nc examples/basic.nc --profile profiles/default_3axis.yaml --out output/basic.xlsx
python -m nc_time_twin estimate --nc examples/basic.nc --profile profiles/default_3axis.yaml --out output/basic.html
python -m nc_time_twin estimate --nc examples/basic.nc --profile profiles/default_3axis.yaml --out output/basic.csv
python -m nc_time_twin estimate --nc examples/basic.nc --profile profiles/default_3axis.yaml --out output/basic.json
```

也可用 `--format` 明確指定：

```powershell
python -m nc_time_twin estimate --nc examples/basic.nc --profile profiles/default_3axis.yaml --out output/report.data --format json
```

支援格式：

- `json`
- `csv`
- `xlsx`
- `excel`
- `html`

## 6. 指定進給單位與嚴格檢查

若 `G21/G94 F` 為 mm/min：

```powershell
python -m nc_time_twin estimate --nc examples/basic.nc --profile profiles/default_3axis.yaml --feed-unit mm_per_min --out output/basic.xlsx
```

若 `G21/G94 F` 為 m/min，例如 `F6` 代表 6000 mm/min：

```powershell
python -m nc_time_twin estimate --nc examples/basic.nc --profile profiles/default_3axis.yaml --feed-unit m_per_min --out output/basic.xlsx
```

若希望自動判斷：

```powershell
python -m nc_time_twin estimate --nc examples/basic.nc --profile profiles/default_3axis.yaml --feed-unit auto --out output/basic.xlsx
```

嚴格檢查：

```powershell
python -m nc_time_twin estimate --nc examples/ToolPathSource.opti.nc --profile profiles/default_3axis.yaml --strict-feed --out output/opti.xlsx
```

若希望 critical feed sanity issue 讓 CLI 回傳失敗：

```powershell
python -m nc_time_twin estimate --nc examples/ToolPathSource.opti.nc --profile profiles/default_3axis.yaml --strict-feed --fail-on-sanity-error --out output/opti.xlsx
```

## 7. 比較優化前後 NC

比較 candidate NC 與 source NC：

```powershell
python -m nc_time_twin estimate --nc examples/ToolPathSource.opti.nc --compare-nc examples/ToolPathSource.nc --profile profiles/default_3axis.yaml --out output/compare.xlsx
```

報表包含：

- source / candidate 總時間。
- 總時間差與切削時間差。
- 幾何是否一致。
- 是否判定為時間退化。
- 各進給區間時間差。
- 最明顯退化 blocks。

若時間退化時要讓 CLI 回傳 exit code 1：

```powershell
python -m nc_time_twin estimate --nc examples/ToolPathSource.opti.nc --compare-nc examples/ToolPathSource.nc --profile profiles/default_3axis.yaml --fail-on-regression --out output/compare.xlsx
```

允許 5% 退化：

```powershell
python -m nc_time_twin estimate --nc candidate.nc --compare-nc source.nc --profile profiles/default_3axis.yaml --max-regression-ratio 0.05 --fail-on-regression --out output/compare.xlsx
```

## 8. 正規化進給值

將 m/min 轉成 mm/min：

```powershell
python -m nc_time_twin normalize-feed --nc examples/ToolPathSource.opti.nc --profile profiles/default_3axis.yaml --input-feed-unit m_per_min --out output/ToolPathSource.normalized.nc --print-summary
```

若輸入本來是 mm/min，但仍希望依 `max_cut_feed_mm_min` 截斷過高 F 值：

```powershell
python -m nc_time_twin normalize-feed --nc input.nc --profile profiles/default_3axis.yaml --input-feed-unit mm_per_min --out output/input.normalized.nc --print-summary
```

注意：

- `normalize-feed` 不允許覆寫原檔。
- 只改寫公制 `G21` 且每分鐘進給 `G94` 狀態下的 F 字。
- `G95`、`G93`、英制 `G20` 狀態會跳過。
- 註解中的 F 不會被改寫。
- 超過 `max_cut_feed_mm_min` 的值會被截斷。

## 9. 第二階段 Benchmark 與 Profile 校正

產生 benchmark NC：

```powershell
python -m nc_time_twin generate-benchmark --profile profiles/default_phase2_3axis.yaml --out output/phase2_benchmark.nc --print-summary
```

校正資料 CSV 欄位：

```text
case_id,nc_file,actual_total_time_sec
case_001,phase2_benchmark.nc,123.45
```

執行校正：

```powershell
python -m nc_time_twin calibrate-profile --dataset output/calibration.csv --profile profiles/default_phase2_3axis.yaml --out profiles/my_machine_phase2.yaml --print-summary
```

若 CSV 中 `nc_file` 為相對路徑，可指定 base dir：

```powershell
python -m nc_time_twin calibrate-profile --dataset output/calibration.csv --profile profiles/default_phase2_3axis.yaml --nc-base-dir output --out profiles/my_machine_phase2.yaml --print-summary
```

## 10. 使用 GUI

啟動：

```powershell
python -m nc_time_twin.gui.main_window
```

或：

```powershell
nc-time-twin-gui
```

若使用 console script 後仍看到舊畫面，請重新安裝目前專案：

```powershell
pip install -e .
```

GUI 操作流程：

1. 在 `估測` 頁選擇 NC-Code。
2. 確認 Machine Profile，預設為 `profiles/default_3axis.yaml`。
3. 可直接設定 CLI 進階估測選項：
   - 比較 NC-Code。
   - Feed Unit。
   - Time Model。
   - Max Regression Ratio。
   - strict-feed。
   - regression / sanity 警示。
4. 按 `Estimate`。
5. 在同一頁查看中文估測摘要、完整 Results 與 Warnings。
6. 到 `Blocks` 頁查看每個 block 的明細。
7. 到 `Charts` 頁查看 XY Toolpath、Block Time；Phase 2 結果會額外顯示 velocity 圖。
8. 若需要 Report，在 `報表輸出` 區選擇輸出資料夾與格式，按 `輸出報表`。

GUI Estimate 後只會自動輸出 log 到 `logs/`，不會自動輸出 Report。Report 由使用者手動決定是否輸出與輸出位置，預設資料夾為 `output/`。

GUI 的 `Tools` 頁包含：

- Feed 正規化。
- Benchmark 產生。
- Profile 校正。

## 11. 報表閱讀方式

### 11.1 Summary

重點欄位：

- `total_time_sec`
- `total_time_text`
- `rapid_time_sec`
- `cutting_time_sec`
- `arc_time_sec`
- `auxiliary_time_sec`
- `total_length_mm`
- `warning_count`
- `feed_unit_effective`
- `feed_capped_count`
- `phase2_segment_count`
- `phase2_junction_count`
- `phase2_bottleneck_count`
- `phase2_dynamic_sample_count`

### 11.2 Blocks

重點欄位：

- `line_no`
- `raw`
- `type`
- `start` / `end`
- `length_mm`
- `estimated_time_sec`
- `feedrate`
- `feed_mode`
- `feed_unit`
- `effective_feed_mm_min`
- `feed_capped`
- `warnings`
- Phase 2 欄位：entry / exit / peak velocity、v/a/j cap、profile type、slowdown ratio、bottleneck reason。

### 11.3 Diagnostics

Excel diagnostics 整合：

- warnings。
- feed histogram。
- top slow blocks。
- feed sanity summary。
- feed sanity issues。
- comparison。
- Phase 2 junctions / bottlenecks。

### 11.4 Log

Log 適合人工審查，包含：

- Summary。
- Warnings。
- Feed Histogram。
- Top Slow Feed Blocks。
- Feed Sanity Summary。
- Feed Sanity Issues。
- Comparison。

## 12. 常見情境

### 12.1 單一 NC 檔加工時間

```powershell
python -m nc_time_twin estimate --nc examples/O0001.NC --profile profiles/default_3axis.yaml --out output/O0001.xlsx
```

### 12.2 使用 Phase 2 估測

```powershell
python -m nc_time_twin estimate --nc examples/basic.nc --profile profiles/default_phase2_3axis.yaml --out output/basic_phase2.xlsx
```

### 12.3 優化後時間反而變長

```powershell
python -m nc_time_twin estimate --nc examples/ToolPathSource.opti.nc --compare-nc examples/ToolPathSource.nc --profile profiles/default_3axis.yaml --fail-on-regression --out output/ToolPathSource_compare.xlsx
```

檢查：

- `comparison.is_regression`
- `comparison.geometry_match`
- `top_time_regression_blocks`
- `feed_sanity_issues`
- `feed_histogram`

### 12.4 懷疑 F 值單位混用

先用嚴格模式估測：

```powershell
python -m nc_time_twin estimate --nc candidate.nc --profile profiles/default_3axis.yaml --strict-feed --out output/candidate.xlsx
```

若報表建議正規化：

```powershell
python -m nc_time_twin normalize-feed --nc candidate.nc --profile profiles/default_3axis.yaml --input-feed-unit m_per_min --out output/candidate.normalized.nc --print-summary
```

### 12.5 校正實際機台 profile

1. 用 `generate-benchmark` 產生測試 NC。
2. 在實機量測加工時間。
3. 建立 CSV。
4. 用 `calibrate-profile` 輸出校正後 profile。
5. 以校正後 profile 重新估測。

## 13. 錯誤與警告處理

- `Missing feedrate F; using default_cut_feed_mm_min`：切削移動缺少 F。
- `G95 requires spindle speed S; using default_cut_feed_mm_min`：G95 缺少主軸轉速。
- `R arc length is approximate in Phase 1`：R 圓弧採近似。
- `Arc radius mismatch`：IJK 圓弧起終半徑差距過大。
- `Suspicious low G94 metric feedrates`：G21/G94 下 F 值偏低，可能是 m/min。
- `Feedrate capped by max_cut_feed_mm_min`：F 超過 profile 上限。
- `reference return time is not estimated`：reference return 設為 `unestimated`。
- Phase 2 bottleneck reason：代表該段受速度、加速度、jerk 或 junction 限制。
