# NC-Time-Twin 參數設定說明

本文件說明會影響估測結果的主要參數，包含 Machine Profile、CLI、GUI、API、輸出欄位、Feed sanity、Normalize-feed 與第二階段 Phase 2 參數。

## 1. Machine Profile YAML

預設 profile：

```text
profiles/default_3axis.yaml
profiles/default_phase2_3axis.yaml
```

建議先複製再修改：

```powershell
Copy-Item profiles/default_phase2_3axis.yaml profiles/my_machine_phase2.yaml
```

### 1.1 頂層參數

| 參數 | 型別 | 限制 | 說明 |
| --- | --- | --- | --- |
| `machine_name` | string | 無 | 機台名稱。 |
| `controller_name` | string | 無 | 控制器名稱。 |
| `kinematic_type` | string | `3_axis` | Phase 2 MVP 目前支援三軸。 |
| `units` | string | `mm` 或 `inch` | profile 預設單位；NC 的 `G20/G21` 仍會覆寫程式狀態。 |
| `feed_unit` | string | `mm_per_min`、`m_per_min`、`inverse_time`、`auto` | G21/G94 F 值解讀方式。 |
| `axes` | object | 必須含 `X/Y/Z` | 各軸速度、加速度、jerk。 |
| `rapid_feed_mm_min` | float | `> 0` | 保留與 Phase 2 校正使用；G00 主要依各軸 rapid velocity。 |
| `max_cut_feed_mm_min` | float | `> 0` | 最大切削進給；估測與 normalize-feed 會套用上限。 |
| `default_cut_feed_mm_min` | float | `> 0` | 缺少或無效 F 時使用。 |
| `default_cut_acc_mm_s2` | float | `> 0` | trapezoid 與 Phase 2 切削加速度基準。 |
| `default_cut_jerk_mm_s3` | float | `> 0` | Phase 2 切削 jerk 基準。 |
| `arc_tolerance_mm` | float | `>= 0` | IJK 圓弧半徑容差。 |
| `arc_chord_tolerance_mm` | float | `> 0` | Phase 2 圓弧離散 chord 容差。 |
| `controller` | object | 見下節 | 控制器、dwell 與 Phase 2 參數。 |
| `event_time` | object | 見下節 | 輔助事件固定時間。 |
| `cycle` | object | 見下節 | canned cycle 參數。 |
| `time_model` | object | 見下節 | 時間模型。 |
| `reference_return` | object | 見下節 | G28/G30 參考點返回估測設定。 |

### 1.2 `feed_unit`

| 值 | 說明 | 適用情境 |
| --- | --- | --- |
| `mm_per_min` | G21/G94 F 視為 mm/min。 | 一般公制每分鐘進給。 |
| `m_per_min` | G21/G94 F 視為 m/min，估測時乘以 1000。 | 優化器輸出 `F6` 代表 6000 mm/min。 |
| `inverse_time` | 將進給視為 inverse time。 | 特殊控制器格式。 |
| `auto` | 自動判斷低 F 是否可能為 m/min。 | 不確定輸出尺度。 |

`auto` 會統計 G21/G94 低 F blocks。若低 F 數量與比例達門檻，會將低於 100 的 F 視為 m/min。

### 1.3 `axes`

| 參數 | 型別 | 限制 | 說明 |
| --- | --- | --- | --- |
| `rapid_velocity_mm_min` | float | `> 0` | G00 同步軸時間使用。 |
| `max_velocity_mm_min` | float | `> 0` | Phase 2 軸向速度限制。 |
| `max_acc_mm_s2` | float | `> 0` | Phase 2 軸向加速度限制。 |
| `max_jerk_mm_s3` | float | `> 0` | Phase 2 軸向 jerk 限制。 |

### 1.4 `controller`

| 參數 | 型別 | 限制 | 說明 |
| --- | --- | --- | --- |
| `interpolation_period_ms` | float | `> 0` | Phase 2 velocity sample 時間基準。 |
| `lookahead_blocks` | int | `>= 0` | 保留給 lookahead。 |
| `junction_tolerance_mm` | float | `>= 0` | Phase 2 junction velocity limit 使用。 |
| `same_direction_angle_threshold_deg` | float | `>= 0` | 判斷近似同向段。 |
| `reverse_angle_threshold_deg` | float | `>= 0` | 判斷反向或急轉角。 |
| `lookahead_max_iterations` | int | `>= 1` | Phase 2 junction velocity propagation 迭代上限。 |
| `velocity_tolerance_mm_s` | float | `> 0` | Phase 2 速度收斂容差。 |
| `phase2_max_samples_per_block` | int | `>= 10` | 每個 block 最多保留的 Phase 2 動態取樣點。 |
| `dwell_p_unit` | string | `ms` 或 `sec` | `G04 P...` 與 G82 `P...` 單位。 |
| `dwell_x_unit` | string | `ms` 或 `sec` | `G04 X...` 單位。 |

### 1.5 `event_time`

| 參數 | 對應事件 | 說明 |
| --- | --- | --- |
| `tool_change_sec` | `M06` | 換刀時間。 |
| `spindle_start_sec` | `M03/M04` | 主軸啟動時間。 |
| `spindle_stop_sec` | `M05` | 主軸停止時間。 |
| `coolant_on_sec` | `M08` | 切削液開啟時間。 |
| `coolant_off_sec` | `M09` | 切削液關閉時間。 |
| `optional_stop_sec` | `M01` | optional stop 時間。 |

### 1.6 `cycle`

| 參數 | 說明 |
| --- | --- |
| `peck_clearance_mm` | G83 啄鑽每次退回後再次接近下一段切削起點的 clearance。 |

### 1.7 `time_model`

| 值 | 說明 |
| --- | --- |
| `constant_velocity` | 長度除以有效進給。 |
| `trapezoid` | 使用 `default_cut_acc_mm_s2` 做簡化加減速。 |
| `phase2` | 使用第二階段三軸 S-curve、axis limit、junction slowdown。 |

CLI 可用 `--time-model` 覆寫 profile：

```powershell
python -m nc_time_twin estimate --nc input.nc --profile profiles/default_3axis.yaml --time-model phase2 --out output/input.xlsx
```

### 1.8 `reference_return`

| 參數 | 說明 |
| --- | --- |
| `mode` | `unestimated`、`fixed`、`rapid`。 |
| `fixed_time_sec` | `mode: fixed` 時使用。 |
| `position` | `mode: rapid` 時使用的 X/Y/Z 參考點位置。 |

## 2. CLI 參數

### 2.1 `estimate`

```powershell
python -m nc_time_twin estimate --nc <NC檔> --profile <profile.yaml> [options]
```

| 參數 | 必填 | 可選值 / 型別 | 說明 |
| --- | --- | --- | --- |
| `--nc` | 是 | path | 要估測的 NC-Code。 |
| `--profile` | 是 | path | machine profile YAML。 |
| `--out` | 否 | path | 指定報表輸出路徑。 |
| `--feed-unit` | 否 | `auto`、`mm_per_min`、`m_per_min`、`inverse_time` | 覆寫 profile feed_unit。 |
| `--time-model` | 否 | `profile`、`constant_velocity`、`trapezoid`、`phase2` | 覆寫 profile time_model.mode。 |
| `--compare-nc` | 否 | path | source/baseline NC-Code。 |
| `--max-regression-ratio` | 否 | float | 允許退化比例，預設 0。 |
| `--fail-on-regression` | 否 | flag | 退化時 exit code 1。 |
| `--strict-feed` | 否 | flag | 啟用嚴格 feed sanity。 |
| `--fail-on-sanity-error` | 否 | flag | feed sanity critical 時 exit code 1。 |
| `--format` | 否 | `json`、`csv`、`xlsx`、`excel`、`html` | 指定輸出格式。 |
| `--print-summary` | 否 | flag | 印出 summary JSON。 |

### 2.2 `normalize-feed`

| 參數 | 必填 | 可選值 / 型別 | 說明 |
| --- | --- | --- | --- |
| `--nc` | 是 | path | 要正規化的 NC。 |
| `--profile` | 是 | path | 讀取 `max_cut_feed_mm_min`。 |
| `--out` | 是 | path | 正規化後 NC，不能與輸入相同。 |
| `--input-feed-unit` | 是 | `mm_per_min`、`m_per_min` | 輸入 F 尺度。 |
| `--print-summary` | 否 | flag | 印出摘要 JSON。 |

### 2.3 `generate-benchmark`

| 參數 | 必填 | 說明 |
| --- | --- | --- |
| `--profile` | 是 | Phase 2 profile YAML。 |
| `--out` | 是 | benchmark NC 輸出路徑。 |
| `--print-summary` | 否 | 印出輸出路徑、machine name、line count。 |

### 2.4 `calibrate-profile`

| 參數 | 必填 | 說明 |
| --- | --- | --- |
| `--dataset` | 是 | CSV，需有 `nc_file` 與 `actual_total_time_sec`，可選 `case_id`。 |
| `--profile` | 是 | Base Phase 2 profile。 |
| `--out` | 是 | 校正後 profile YAML。 |
| `--nc-base-dir` | 否 | dataset 中相對 `nc_file` 的基準資料夾。 |
| `--print-summary` | 否 | 印出校正摘要 JSON。 |

## 3. GUI 欄位

### 3.1 估測頁

| 欄位 / 控制項 | 說明 |
| --- | --- |
| `NC-Code` | 欲估測的 NC-Code。 |

| `Feed Unit` | 覆寫 feed unit；空值代表使用 profile。 |
| `Time Model` | 覆寫 time model；空值代表使用 profile。 |
| `Max Regression Ratio` | comparison 退化容許比例。 |

| `顯示回歸失敗警示` | 顯示 regression 警示，不中止 GUI 結果。 |
| `顯示 Feed sanity 失敗警示` | 顯示 feed sanity critical 警示，不中止 GUI 結果。 |
| `Estimate` | 執行估測，只自動產生 log。 |
| `估測摘要` | 中文總覽。 |
| `完整 Results` | summary key-value。 |
| `Warnings` | warning 與 GUI 警示。 |

### 3.2 Tools 頁

| 工具 | 欄位 | 說明 |
| --- | --- | --- |

## 6. Feed sanity 參數與規則

| 常數 | 值 | 說明 |
| --- | --- | --- |
| `LOW_EFFECTIVE_FEED_MM_MIN` | `1000.0` | 有效進給低於此值列為低速風險。 |
| `VERY_LOW_EFFECTIVE_FEED_MM_MIN` | `500.0` | 更嚴重低速。 |
| `LOW_RAW_G94_FEED` | `100.0` | G21/G94 原始 F 低於此值可能是 m/min。 |
| `MID_RAW_G94_FEED` | `1000.0` | 100 到 999 的原始 F 用於混合尺度判斷。 |
| `EXTREME_RAW_MULTIPLIER` | `5.0` | 極端原始 F 門檻為 `max_cut_feed_mm_min * 5`。 |

## 10. 建議設定流程

1. 從 `profiles/default_phase2_3axis.yaml` 複製實際機台 profile。
2. 先設定 X/Y/Z 的 rapid velocity、max velocity、max acc、max jerk。
3. 設定 `max_cut_feed_mm_min` 與 `default_cut_feed_mm_min`。
4. 設定 `default_cut_acc_mm_s2` 與 `default_cut_jerk_mm_s3`。
5. 填入 `event_time`。
6. 確認 dwell 單位。
7. 使用 `generate-benchmark` 產生 benchmark NC。
8. 實機量測後建立 calibration CSV。
9. 使用 `calibrate-profile` 輸出校正後 profile。
10. 以校正後 profile 執行 `--time-model phase2` 或直接設定 `time_model.mode: phase2`。
