# NC-Time-Twin 參數設定說明

本文件說明專案中可由使用者設定或會影響結果的參數。範圍包含：

- Machine profile YAML 參數。
- CLI 參數。
- GUI 欄位。
- API 參數。
- 輸出報表欄位。
- Feed sanity 與 normalize-feed 相關參數。

## 1. Machine Profile YAML

預設檔案為：

```text
profiles/default_3axis.yaml
```

Machine profile 由 `MachineProfile` Pydantic schema 驗證。建議以預設檔複製後修改：

```powershell
Copy-Item profiles/default_3axis.yaml profiles/my_machine.yaml
```

### 1.1 頂層參數

| 參數 | 型別 | 預設值 | 限制 | 說明 |
| --- | --- | --- | --- | --- |
| `machine_name` | string | `"Default 3-Axis CNC"` | 無 | 機台名稱，主要用於識別。 |
| `controller_name` | string | `"generic"` | 無 | 控制器名稱，主要用於識別。 |
| `units` | string | `"mm"` | `mm` 或 `inch` | profile 預設單位。NC 中 `G20/G21` 仍會影響程式行的單位。 |
| `feed_unit` | string | `"mm_per_min"`；預設 YAML 為 `"auto"` | `mm_per_min`、`m_per_min`、`inverse_time`、`auto` | 決定 G21/G94 F 值的解讀方式。 |
| `axes` | object | 必填 | 必須含 `X`、`Y`、`Z` | 各軸速度、加速度與 jerk。 |
| `rapid_feed_mm_min` | float | `12000.0`；預設 YAML 為 `10000` | `> 0` | 保留參數；目前 rapid time 實際使用各軸 `rapid_velocity_mm_min`。 |
| `max_cut_feed_mm_min` | float | `10000.0` | `> 0` | 最大切削進給。超過時估測會截斷，normalize-feed 也會套用上限。 |
| `default_cut_feed_mm_min` | float | `1000.0` | `> 0` | 缺少或無效 F 值時使用的預設切削進給。 |
| `default_cut_acc_mm_s2` | float | `800.0` | `> 0` | trapezoid 時間模型使用的切削加速度。 |
| `arc_tolerance_mm` | float | `0.01` | `>= 0` | IJK 圓弧起點半徑與終點半徑允許差異，超過會警告。 |
| `controller` | object | 見下節 |  | 控制器與 dwell 相關設定。 |
| `event_time` | object | 見下節 |  | 輔助事件固定時間。 |
| `cycle` | object | 見下節 |  | canned cycle 參數。 |
| `time_model` | object | 見下節 |  | 切削時間模型。 |
| `reference_return` | object | 見下節 |  | G28/G30 參考點返回估測設定。 |

### 1.2 `feed_unit`

| 值 | 說明 | 適用情境 |
| --- | --- | --- |
| `mm_per_min` | G21/G94 F 值視為 mm/min。 | 一般 Fanuc 類公制每分鐘進給。 |
| `m_per_min` | G21/G94 F 值視為 m/min，估測時乘以 1000。 | 優化器或後處理輸出以 m/min 表示，例如 `F6` 代表 6000 mm/min。 |
| `inverse_time` | 將進給視為 inverse time。 | 特殊程式或控制器格式。 |
| `auto` | 自動判斷低 F 值是否可能代表 m/min。 | 不確定優化輸出尺度時。 |

`auto` 判斷規則目前為：

- 蒐集有 F 值的 linear / arc blocks。
- 找出 G21/G94 且 `0 < F < 100` 的 blocks。
- 若低 F blocks 數量至少 20，且占所有 feed blocks 比例至少 5%，則有效單位設為 `auto_m_per_min`。
- `auto_m_per_min` 下，只有低於 100 的 G21/G94 F 值會乘以 1000。
- 否則使用 `mm_per_min`。

### 1.3 `axes`

每個軸使用 `AxisProfile`：

| 參數 | 型別 | 預設值 | 限制 | 說明 |
| --- | --- | --- | --- | --- |
| `rapid_velocity_mm_min` | float | `12000.0` | `> 0` | 該軸快速移動速度。G00 時用於同步軸時間計算。 |
| `max_velocity_mm_min` | float | `12000.0` | `> 0` | 該軸最大速度；目前保留，未直接用於時間估測。 |
| `max_acc_mm_s2` | float | `1000.0` | `> 0` | 該軸最大加速度；目前 rapid 未使用軸加速度模型。 |
| `max_jerk_mm_s3` | float | `10000.0` | `> 0` | 該軸 jerk；目前保留。 |

預設 YAML：

```yaml
axes:
  X:
    rapid_velocity_mm_min: 10000
    max_velocity_mm_min: 12000
    max_acc_mm_s2: 1000
    max_jerk_mm_s3: 10000
  Y:
    rapid_velocity_mm_min: 10000
    max_velocity_mm_min: 12000
    max_acc_mm_s2: 1000
    max_jerk_mm_s3: 10000
  Z:
    rapid_velocity_mm_min: 5000
    max_velocity_mm_min: 10000
    max_acc_mm_s2: 800
    max_jerk_mm_s3: 8000
```

### 1.4 `controller`

| 參數 | 型別 | 預設值 | 限制 | 說明 |
| --- | --- | --- | --- | --- |
| `interpolation_period_ms` | float | `2.0` | `> 0` | 插補週期；目前保留，尚未直接用於時間估測。 |
| `lookahead_blocks` | int | `100` | `>= 0` | lookahead blocks；目前保留。 |
| `junction_tolerance_mm` | float | `0.01` | `>= 0` | 轉角容差；目前保留。 |
| `dwell_p_unit` | string | `"ms"` | `ms` 或 `sec` | `G04 P...` 與 G82 `P...` 的時間單位。 |
| `dwell_x_unit` | string | `"sec"` | `ms` 或 `sec` | `G04 X...` 的時間單位。 |

範例：

```yaml
controller:
  interpolation_period_ms: 2
  lookahead_blocks: 100
  junction_tolerance_mm: 0.01
  dwell_p_unit: "ms"
  dwell_x_unit: "sec"
```

### 1.5 `event_time`

| 參數 | 型別 | 預設值 | 限制 | 對應事件 |
| --- | --- | --- | --- | --- |
| `tool_change_sec` | float | `8.0` | `>= 0` | `M06` |
| `spindle_start_sec` | float | `2.0` | `>= 0` | `M03`、`M04` |
| `spindle_stop_sec` | float | `1.0` | `>= 0` | `M05` |
| `coolant_on_sec` | float | `0.5` | `>= 0` | `M08` |
| `coolant_off_sec` | float | `0.5` | `>= 0` | `M09` |
| `optional_stop_sec` | float | `0.0` | `>= 0` | `M01` |

這些時間會直接加入總時間，並彙整到 `auxiliary_time_sec`。

### 1.6 `cycle`

| 參數 | 型別 | 預設值 | 限制 | 說明 |
| --- | --- | --- | --- | --- |
| `peck_clearance_mm` | float | `1.0` | `>= 0` | G83 啄鑽每次退回後再次接近下一段切削起點時的 clearance。 |

### 1.7 `time_model`

| 參數 | 型別 | 預設值 | 限制 | 說明 |
| --- | --- | --- | --- | --- |
| `mode` | string | `"constant_velocity"` | `constant_velocity` 或 `trapezoid` | 切削移動時間模型。 |

`constant_velocity`：

```text
time_sec = length_mm / (feed_mm_min / 60)
```

`trapezoid`：

- 使用 `default_cut_acc_mm_s2` 做加減速估算。
- 長距離使用加速、等速、減速。
- 短距離使用三角速度曲線。

### 1.8 `reference_return`

| 參數 | 型別 | 預設值 | 限制 | 說明 |
| --- | --- | --- | --- | --- |
| `mode` | string | `"unestimated"` | `unestimated`、`fixed`、`rapid` | G28/G30 時間估測模式。 |
| `fixed_time_sec` | float | `0.0` | `>= 0` | `mode: fixed` 時使用的固定時間。 |
| `position` | object | X/Y/Z 都為 `0.0` | 軸名稱會轉大寫 | 參考點位置。 |

`mode` 說明：

- `unestimated`：時間設為 0，產生 warning。
- `fixed`：時間使用 `fixed_time_sec`。
- `rapid`：依起點、指定軸、`position` 與各軸 `rapid_velocity_mm_min` 估算。

範例：

```yaml
reference_return:
  mode: "rapid"
  fixed_time_sec: 0.0
  position:
    X: 0.0
    Y: 0.0
    Z: 0.0
```

## 2. CLI 參數

CLI 主程式：

```powershell
python -m nc_time_twin <command> [options]
```

或安裝後：

```powershell
nc-time-twin <command> [options]
```

### 2.1 `estimate`

```powershell
python -m nc_time_twin estimate --nc <NC檔> --profile <profile.yaml> [options]
```

| 參數 | 必填 | 可選值 / 型別 | 說明 |
| --- | --- | --- | --- |
| `--nc` | 是 | path | 要估測的 NC-Code 檔案。 |
| `--profile` | 是 | path | machine profile YAML。 |
| `--out` | 否 | path | 指定報表輸出路徑。省略時會印出 summary JSON。 |
| `--feed-unit` | 否 | `auto`、`mm_per_min`、`m_per_min`、`inverse_time` | 覆寫 profile 中的 `feed_unit`。 |
| `--compare-nc` | 否 | path | source/baseline NC-Code。指定後會執行 candidate vs source 比較。 |
| `--max-regression-ratio` | 否 | float，預設 `0.0` | 判斷退化時允許的比例。 |
| `--fail-on-regression` | 否 | flag | 若 geometry match 且退化比例超過上限，exit code 回傳 1。 |
| `--strict-feed` | 否 | flag | 若未指定 `--feed-unit`，會以 `mm_per_min` 明確估測，且 feed sanity 對低有效進給更嚴格。 |
| `--fail-on-sanity-error` | 否 | flag | 若 feed sanity 有 critical issue，exit code 回傳 1。 |
| `--format` | 否 | `json`、`csv`、`xlsx`、`excel`、`html` | 指定輸出格式；省略時由 `--out` 副檔名推論。 |
| `--print-summary` | 否 | flag | 將 summary JSON 印到 stdout。 |

### 2.2 `normalize-feed`

```powershell
python -m nc_time_twin normalize-feed --nc <NC檔> --profile <profile.yaml> --out <輸出NC檔> --input-feed-unit <單位>
```

| 參數 | 必填 | 可選值 / 型別 | 說明 |
| --- | --- | --- | --- |
| `--nc` | 是 | path | 要正規化的 NC-Code 檔案。 |
| `--profile` | 是 | path | machine profile YAML，用於讀取 `max_cut_feed_mm_min`。 |
| `--out` | 是 | path | 正規化後 NC-Code 輸出路徑，不能與輸入檔相同。 |
| `--input-feed-unit` | 是 | `mm_per_min`、`m_per_min` | 輸入 NC 中 G21/G94 F 值的原始尺度。 |
| `--print-summary` | 否 | flag | 印出正規化摘要 JSON。 |

### 5.4 Feed histogram

有效進給區間固定為：

- `<1000`
- `1000-2999`
- `3000-4499`
- `4500-5999`
- `>=6000`

每個區間包含：

- `effective_feed_band_mm_min`
- `block_count`
- `length_mm`
- `time_sec`
- `raw_feed_min`
- `raw_feed_max`
- `effective_feed_min`
- `effective_feed_max`

### 5.5 Top slow blocks

預設最多列出 20 個 blocks。若有有效進給低於 1000 mm/min 的 blocks，會優先從這些低速 blocks 中依時間排序；否則從所有 feed blocks 中依時間排序。

欄位包含：

- `line_no`
- `type`
- `length_mm`
- `estimated_time_sec`
- `feedrate`
- `feed_unit`
- `effective_feed_mm_min`
- `feed_capped`
- `raw`

## 6. Feed sanity 參數與規則

目前 feed sanity 常數如下：

| 常數 | 值 | 說明 |
| --- | --- | --- |
| `LOW_EFFECTIVE_FEED_MM_MIN` | `1000.0` | 有效進給低於此值會列為低速風險。 |
| `VERY_LOW_EFFECTIVE_FEED_MM_MIN` | `500.0` | 有效進給低於此值會更嚴重。 |
| `LOW_RAW_G94_FEED` | `100.0` | G21/G94 原始 F 低於此值可能是 m/min 尺度。 |
| `MID_RAW_G94_FEED` | `1000.0` | 100 到 999 的 G21/G94 原始 F 用於判斷混合尺度。 |
| `EXTREME_RAW_MULTIPLIER` | `5.0` | 極端原始 F 門檻為 `max_cut_feed_mm_min * 5`。 |

Feed sanity summary 欄位：

| 欄位 | 說明 |
| --- | --- |
| `feed_sanity_issue_count` | issue 總數。 |
| `feed_sanity_critical_count` | critical issue 數。 |
| `feed_sanity_warning_count` | warning issue 數。 |
| `feed_sanity_low_effective_count` | 有效進給低於 1000 mm/min 的 blocks。 |
| `feed_sanity_low_raw_g94_count` | G21/G94 原始 F 低於 100 的 blocks。 |
| `feed_sanity_mid_raw_g94_count` | G21/G94 原始 F 介於 100 到 999 的 blocks。 |
| `feed_sanity_extreme_raw_count` | 原始 F 高於 `max_cut_feed_mm_min * 5` 的 blocks。 |
| `feed_sanity_extreme_raw_threshold` | 極端原始 F 門檻值。 |
| `feed_sanity_strict_feed` | 是否使用 strict feed 模式。 |

Feed sanity issue 欄位：

| 欄位 | 說明 |
| --- | --- |
| `severity` | `warning` 或 `critical`。 |
| `code` | issue 類型，例如 `mixed_feed_scale`、`low_effective_feed`、`extreme_raw_feed`。 |
| `line_no` | 對應 NC 行號；聚合型 issue 為 null。 |
| `feedrate` | 原始 F 值。 |
| `effective_feed_mm_min` | 有效進給。 |
| `estimated_time_sec` | 該 block 估測時間。 |
| `raw` | 原始 NC 行。 |
| `message` | 診斷訊息。 |
| `recommendation` | 建議處置。 |
| `sample_lines` | 聚合型 issue 的範例行號。 |

## 7. Comparison 參數

比較結果欄位：

| 欄位 | 說明 |
| --- | --- |
| `source_label` | source NC 路徑標籤。 |
| `candidate_label` | candidate NC 路徑標籤。 |
| `block_count_match` | block 數是否一致。 |
| `geometry_match` | block 類型、長度、起點、終點是否一致。 |
| `is_regression` | geometry match 且退化比例超過上限。 |
| `max_regression_ratio` | 使用者設定的允許退化比例。 |
| `regression_ratio` | `(candidate_total - source_total) / source_total`。 |
| `total_time_delta_sec` | candidate 總時間減 source 總時間。 |
| `cutting_time_delta_sec` | candidate 切削時間減 source 切削時間。 |
| `source_total_time_sec` | source 總秒數。 |
| `candidate_total_time_sec` | candidate 總秒數。 |
| `source_total_time_text` | source HH:MM:SS。 |
| `candidate_total_time_text` | candidate HH:MM:SS。 |
| `feed_band_deltas` | 依 candidate 有效進給區間彙整的時間差。 |
| `top_time_regression_blocks` | 時間差最大的 blocks。 |

## 8. Normalize-feed summary 參數

`normalize-feed --print-summary` 輸出：

| 欄位 | 說明 |
| --- | --- |
| `input_path` | 輸入 NC 路徑。 |
| `output_path` | 輸出 NC 路徑。 |
| `input_feed_unit` | 輸入 F 值尺度。 |
| `output_feed_unit` | 輸出尺度，固定為 `mm_per_min`。 |
| `rewritten_feed_count` | 改寫的 F 字數量。 |
| `capped_feed_count` | 改寫時被 `max_cut_feed_mm_min` 截斷的數量。 |
| `skipped_feed_count` | 因 G20/G93/G95 等狀態而跳過的 F 字數量。 |
| `max_cut_feed_mm_min` | 套用的最大切削進給。 |
| `changed_lines` | 有被改寫的原始行號。 |

## 9. 建議設定流程

1. 從 `profiles/default_3axis.yaml` 複製一份實際機台 profile。
2. 先校正 X/Y/Z 的 `rapid_velocity_mm_min`。
3. 設定 `max_cut_feed_mm_min` 為機台或製程允許上限。
4. 設定 `default_cut_feed_mm_min` 為缺少 F 值時可接受的保守值。
5. 依現場經驗填入 `event_time`。
6. 若 NC 常使用 `G04 P...`，確認 `dwell_p_unit` 是 `ms` 或 `sec`。
7. 若常處理優化器輸出，先使用 `feed_unit: "auto"`，再搭配 `--strict-feed` 檢查。
8. 若已確定後處理器輸出單位，改用明確的 `mm_per_min` 或 `m_per_min`。
9. 若要做優化前後驗證，搭配 `--compare-nc` 與 `--fail-on-regression`。
