以下針對「第一階段：看得懂 NC-Code 的加工時間估算器」設計可直接實作的演算法。重點是先把 NC-Code 解析、模態狀態、IR 中介模型、幾何路徑、簡化機台運動學與時間估算做穩。S 曲線與前瞻控制在第一階段先保留介面，不建議一開始就完整實作，否則容易讓開發範圍失控。

第一階段本質如下：

```
NC-Code → 解析成結構化指令 → 更新 CNC 模態狀態 → 轉成 IR → 計算路徑長度 → 套用簡化機台模型 → 輸出時間估測
```

---

# 一、第一階段演算法總流程

## Algorithm 0：主流程 Pipeline

```python
function estimate_nc_time(nc_file_path, machine_profile_path):

    # 1. 載入機台設定
    machine_profile = load_machine_profile(machine_profile_path)

    # 2. 讀取 NC-Code
    raw_lines = read_file(nc_file_path)

    # 3. 前處理：移除註解、空白、行號
    clean_lines = preprocess_nc_lines(raw_lines)

    # 4. 建立初始 CNC 模態狀態
    state = init_modal_state()

    # 5. 建立巨集變數表
    macro_table = {}

    # 6. 建立 IR Program
    ir_program = []

    # 7. 逐行解析
    for line_no, raw_line in clean_lines:

        # 7.1 處理 # 變數，例如 #101=8000
        if is_macro_assignment(raw_line):
            update_macro_table(raw_line, macro_table)
            continue

        # 7.2 將 #101、#102 替換成實際數值
        expanded_line = expand_macro_variables(raw_line, macro_table)

        # 7.3 Tokenize，例如 G01 X10.0 Y20.0 F800
        tokens = tokenize(expanded_line)

        # 7.4 更新模態狀態，例如 G90/G91/G17/G18/G19/F/S
        prev_state = copy(state)
        update_modal_state(state, tokens)

        # 7.5 根據 tokens + state 產生 IR block
        blocks = build_ir_blocks(
            line_no=line_no,
            raw=raw_line,
            tokens=tokens,
            prev_state=prev_state,
            state=state,
            machine_profile=machine_profile
        )

        # 7.6 加入 IR Program
        ir_program.extend(blocks)

    # 8. 計算每個 IR block 的幾何長度
    for block in ir_program:
        compute_geometry(block, machine_profile)

    # 9. 計算每個 block 的估測時間
    for block in ir_program:
        estimate_block_time(block, machine_profile)

    # 10. 統計結果
    result = summarize_result(ir_program)

    # 11. 輸出給 GUI / Report
    return result
```

這個主流程要注意一件事：第一階段雖然還不是完整數位雙生，但資料流一定要像「虛擬控制器」一樣設計。也就是每一行 NC-Code 都不能只當文字看，而是要根據目前 CNC 狀態解讀。

# 二、NC-Code 前處理演算法

NC-Code 會有註解、行號、空白、百分比符號、程式號、巨集變數等。第一階段要先把它整理成乾淨指令。

## Algorithm 1：NC-Code 前處理

```python
function preprocess_nc_lines(raw_lines):

    clean_lines = []

    for index, line in enumerate(raw_lines):

        original_line = line

        # 1. 去除左右空白
        line = strip(line)

        # 2. 忽略空行
        if line == "":
            continue

        # 3. 忽略 % 符號
        if line == "%":
            continue

        # 4. 移除括號註解，例如 (TOOL DIA 63.0)
        line = remove_parentheses_comment(line)

        # 5. 移除分號後註解，例如 G01 X10 ; comment
        line = remove_semicolon_comment(line)

        # 6. 移除行號，例如 N100 G01 X10
        line = remove_sequence_number(line)

        # 7. 統一轉大寫
        line = uppercase(line)

        # 8. 再次去除空白
        line = normalize_spaces(line)

        # 9. 若處理後為空，跳過
        if line == "":
            continue

        clean_lines.append((index + 1, line))

    return clean_lines
```

註解移除要小心，不要把括號內的刀具資訊全部丟掉後就完全不可用。第一階段可以先忽略註解，但建議保留原始 raw line，未來可以從註解中抓刀具名稱、刀徑、加工工序名稱。

# 三、巨集變數解析演算法

很多 NC-Code 會出現：

```
#101=8000
#102=900
#103=930

G01 X100. F#103
```

如果不處理 `#103`，進給率就解析不到。

## Algorithm 2：巨集變數處理

```python
function is_macro_assignment(line):

    # 支援格式：
    # #101=8000
    # #101 = 8000
    # #101=930.5

    if line matches r"#\d+\s*=\s*[-+]?\d+(\.\d+)?":
        return True
    else:
        return False
```

```python
function update_macro_table(line, macro_table):

    variable_id = extract_variable_id(line)    # 例如 101
    value = extract_value(line)                # 例如 8000

    macro_table[variable_id] = value
```

```python
function expand_macro_variables(line, macro_table):

    # 尋找所有 #數字，例如 #101, #102
    variables = find_all_macro_refs(line)

    for var in variables:

        var_id = get_variable_id(var)

        if var_id exists in macro_table:
            line = replace(line, var, macro_table[var_id])
        else:
            add_warning("Unknown macro variable: " + var)
            line = replace(line, var, "UNKNOWN")

    return line
```

第一階段只建議支援「簡單變數指定與替換」。複雜巨集，例如 `IF`、`WHILE`、`GOTO`、條件跳躍，不建議第一階段完整處理，遇到時先警告。

# 四、Tokenizer 詞法解析演算法

Tokenizer 的任務是把一行 NC-Code 拆成結構化 token。

例如：

```
G01 X10.0 Y20.0 Z-5.0 F800
```

轉成：

```python
{
    "G": [1],
    "X": 10.0,
    "Y": 20.0,
    "Z": -5.0,
    "F": 800.0
}
```

## Algorithm 3：Tokenizer

```python
function tokenize(line):

    tokens = empty dictionary

    # 1. 移除空白
    line = remove_spaces(line)

    # 2. 找出所有 Letter + Number
    # 例如 G01, X10.5, Y-20.0, F800
    matches = regex_find_all(
        pattern = r"([A-Z])([-+]?\d+(\.\d+)?)",
        text = line
    )

    for match in matches:

        letter = match.group(1)
        value = parse_float(match.group(2))

        # G/M 可能同一行有多個，例如 G90 G54 G17
        if letter in ["G", "M"]:
            if letter not in tokens:
                tokens[letter] = []
            tokens[letter].append(int(value))

        else:
            tokens[letter] = value

    # 3. 檢查是否有無法解析的殘留字元
    unknown_parts = find_unparsed_parts(line, matches)
    if unknown_parts not empty:
        tokens["WARNINGS"].append("Unparsed content: " + unknown_parts)

    return tokens
```

注意：`G01`、`G1`、`G001` 都應該視為 `G1`。M-code 也一樣。

# 五、Modal State Machine 模態狀態機

這是第一階段最重要的部分。CNC 指令有「狀態延續」特性，不能只看單行。

例如：

```
G90 G54 G17
G01 X10 Y10 F500
X20
Y30
```

後兩行沒有寫 G01，但仍然是 G01。

## Algorithm 4：初始化 Modal State

```python
function init_modal_state():

    state = ModalState()

    state.motion = None              # G00/G01/G02/G03
    state.plane = "G17"              # XY plane
    state.distance_mode = "G90"      # absolute
    state.unit = "mm"                # G21
    state.feed_mode = "G94"          # mm/min
    state.feedrate = None
    state.spindle_speed = None
    state.current_position = (0, 0, 0)
    state.current_tool = None
    state.coolant_on = False
    state.spindle_on = False

    return state
```

## Algorithm 5：更新 Modal State

```python
function update_modal_state(state, tokens):

    # 1. 處理 G-code
    if "G" in tokens:

        for g in tokens["G"]:

            if g == 0:
                state.motion = "G00"

            elif g == 1:
                state.motion = "G01"

            elif g == 2:
                state.motion = "G02"

            elif g == 3:
                state.motion = "G03"

            elif g == 17:
                state.plane = "G17"

            elif g == 18:
                state.plane = "G18"

            elif g == 19:
                state.plane = "G19"

            elif g == 20:
                state.unit = "inch"

            elif g == 21:
                state.unit = "mm"

            elif g == 90:
                state.distance_mode = "G90"

            elif g == 91:
                state.distance_mode = "G91"

            elif g == 93:
                state.feed_mode = "G93"     # inverse time，第一階段可先警告

            elif g == 94:
                state.feed_mode = "G94"     # mm/min

            elif g == 95:
                state.feed_mode = "G95"     # mm/rev

            elif g == 4:
                # G04 不是持續運動模態，後續由 IR builder 處理
                pass

            else:
                add_warning("Unsupported G-code: G" + g)

    # 2. 處理 F 進給率
    if "F" in tokens:
        state.feedrate = tokens["F"]

    # 3. 處理 S 主軸轉速
    if "S" in tokens:
        state.spindle_speed = tokens["S"]

    # 4. 處理 T 刀具號
    if "T" in tokens:
        state.current_tool = int(tokens["T"])

    # 5. 處理 M-code
    if "M" in tokens:

        for m in tokens["M"]:

            if m == 3 or m == 4:
                state.spindle_on = True

            elif m == 5:
                state.spindle_on = False

            elif m == 8:
                state.coolant_on = True

            elif m == 9:
                state.coolant_on = False

            elif m == 6:
                # 換刀本身由 IR builder 產生 ToolChange block
                pass
```

# 六、座標解析演算法

NC-Code 通常只寫有變動的軸。例如：

```
G01 X10 Y20 Z-5
X30
```

第二行的 Y/Z 要沿用上一行。

## Algorithm 6：解析終點座標

```python
function resolve_target_position(tokens, state):

    current_x, current_y, current_z = state.current_position

    # 1. 取得輸入座標；若沒有寫，就沿用目前座標
    input_x = tokens.get("X", None)
    input_y = tokens.get("Y", None)
    input_z = tokens.get("Z", None)

    # 2. 單位轉換
    if state.unit == "inch":
        if input_x is not None: input_x = input_x * 25.4
        if input_y is not None: input_y = input_y * 25.4
        if input_z is not None: input_z = input_z * 25.4

    # 3. 絕對座標 G90
    if state.distance_mode == "G90":

        target_x = input_x if input_x is not None else current_x
        target_y = input_y if input_y is not None else current_y
        target_z = input_z if input_z is not None else current_z

    # 4. 相對座標 G91
    else if state.distance_mode == "G91":

        target_x = current_x + input_x if input_x is not None else current_x
        target_y = current_y + input_y if input_y is not None else current_y
        target_z = current_z + input_z if input_z is not None else current_z

    return (target_x, target_y, target_z)
```

# 七、IR 中介模型產生演算法

第一階段建議至少建立這幾種 IR block：

```
RapidMove      G00
LinearMove     G01
ArcMove        G02/G03
Dwell          G04
ToolChange     M06
SpindleEvent   M03/M04/M05
CoolantEvent   M08/M09
ProgramEnd     M30
UnknownBlock   不支援指令
```

## Algorithm 7：由 tokens 建立 IR Block

```python
function build_ir_blocks(line_no, raw, tokens, prev_state, state, machine_profile):

    blocks = []

    # 1. 處理 M-code 事件
    if "M" in tokens:

        for m in tokens["M"]:

            if m == 6:
                block = ToolChangeBlock(
                    line_no=line_no,
                    raw=raw,
                    tool_id=state.current_tool
                )
                blocks.append(block)

            elif m == 3 or m == 4:
                block = SpindleEventBlock(
                    line_no=line_no,
                    raw=raw,
                    event="spindle_start",
                    spindle_speed=state.spindle_speed
                )
                blocks.append(block)

            elif m == 5:
                block = SpindleEventBlock(
                    line_no=line_no,
                    raw=raw,
                    event="spindle_stop"
                )
                blocks.append(block)

            elif m == 8:
                block = CoolantEventBlock(
                    line_no=line_no,
                    raw=raw,
                    event="coolant_on"
                )
                blocks.append(block)

            elif m == 9:
                block = CoolantEventBlock(
                    line_no=line_no,
                    raw=raw,
                    event="coolant_off"
                )
                blocks.append(block)

            elif m == 30:
                block = ProgramEndBlock(
                    line_no=line_no,
                    raw=raw
                )
                blocks.append(block)

    # 2. 處理 G04 暫停
    if contains_gcode(tokens, 4):

        dwell_time = parse_dwell_time(tokens, machine_profile)

        block = DwellBlock(
            line_no=line_no,
            raw=raw,
            dwell_time_sec=dwell_time
        )
        blocks.append(block)

        return blocks

    # 3. 判斷這一行是否有座標移動
    has_axis_word = any key in tokens for key in ["X", "Y", "Z"]

    if has_axis_word and state.motion is not None:

        start_pos = prev_state.current_position
        end_pos = resolve_target_position(tokens, prev_state)

        # G00 快速移動
        if state.motion == "G00":
            block = RapidMoveBlock(
                line_no=line_no,
                raw=raw,
                start=start_pos,
                end=end_pos
            )

        # G01 直線切削
        elif state.motion == "G01":
            block = LinearMoveBlock(
                line_no=line_no,
                raw=raw,
                start=start_pos,
                end=end_pos,
                feedrate=state.feedrate,
                feed_mode=state.feed_mode,
                spindle_speed=state.spindle_speed
            )

        # G02/G03 圓弧
        elif state.motion == "G02" or state.motion == "G03":

            block = ArcMoveBlock(
                line_no=line_no,
                raw=raw,
                direction=state.motion,
                plane=state.plane,
                start=start_pos,
                end=end_pos,
                ijk=(tokens.get("I"), tokens.get("J"), tokens.get("K")),
                r=tokens.get("R"),
                feedrate=state.feedrate,
                feed_mode=state.feed_mode,
                spindle_speed=state.spindle_speed
            )

        blocks.append(block)

        # 更新目前位置
        state.current_position = end_pos

    return blocks
```

這裡有一個關鍵：建立 IR 時要同時保留 `raw` 原始 NC-Code。未來 GUI 顯示分段時間時，可以直接對應回原始行號與原始指令。

# 八、G04 暫停時間解析演算法

不同控制器對 G04 的 P 單位可能不同，有些是毫秒，有些是秒。因此第一階段要放到 Machine Profile 裡設定。

## Algorithm 8：G04 Dwell 解析

```python
function parse_dwell_time(tokens, machine_profile):

    # 常見格式：
    # G04 P1000
    # G04 X2.0

    if "P" in tokens:

        p_value = tokens["P"]

        if machine_profile.controller.dwell_p_unit == "ms":
            return p_value / 1000.0

        elif machine_profile.controller.dwell_p_unit == "sec":
            return p_value

        else:
            add_warning("Unknown dwell P unit")
            return p_value

    elif "X" in tokens:

        x_value = tokens["X"]

        if machine_profile.controller.dwell_x_unit == "sec":
            return x_value

        elif machine_profile.controller.dwell_x_unit == "ms":
            return x_value / 1000.0

    else:
        add_warning("G04 without P or X value")
        return 0.0
```

# 九、進給率換算演算法

G94 是 mm/min，最直觀。

G95 是 mm/rev，需要主軸轉速 S。

G93 是 inverse time feed，第一階段可先支援基本換算或先警告。

## Algorithm 9：取得實際進給速度

```python
function resolve_feedrate_mm_per_min(block, machine_profile):

    if block.feed_mode == "G94":

        if block.feedrate is None:
            block.warning.append("Missing feedrate F")
            return machine_profile.default_cut_feed_mm_min

        return block.feedrate

    elif block.feed_mode == "G95":

        if block.feedrate is None:
            block.warning.append("Missing feedrate F in G95")
            return machine_profile.default_cut_feed_mm_min

        if block.spindle_speed is None:
            block.warning.append("G95 requires spindle speed S")
            return machine_profile.default_cut_feed_mm_min

        # mm/min = mm/rev × rev/min
        return block.feedrate * block.spindle_speed

    elif block.feed_mode == "G93":

        # G93 的 F 通常代表該段 inverse time
        # 第一階段建議先不完整支援
        block.warning.append("G93 inverse time feed is not fully supported in Phase 1")
        return machine_profile.default_cut_feed_mm_min

    else:
        block.warning.append("Unknown feed mode")
        return machine_profile.default_cut_feed_mm_min
```

# 十、幾何路徑計算演算法

## Algorithm 10：直線長度

```python
function compute_line_length(start, end):

    dx = end.x - start.x
    dy = end.y - start.y
    dz = end.z - start.z

    length = sqrt(dx*dx + dy*dy + dz*dz)

    return length
```

---

## Algorithm 11：G00 快速移動軸向距離

G00 不建議直接用三維距離除以快速進給。很多機台的快速移動比較接近「各軸同時移動，各軸速度受各自上限限制」，因此時間可用各軸時間取最大值。

```python
function compute_rapid_time(block, machine_profile):

    dx = abs(block.end.x - block.start.x)
    dy = abs(block.end.y - block.start.y)
    dz = abs(block.end.z - block.start.z)

    vx = machine_profile.axes["X"].rapid_velocity_mm_min / 60.0
    vy = machine_profile.axes["Y"].rapid_velocity_mm_min / 60.0
    vz = machine_profile.axes["Z"].rapid_velocity_mm_min / 60.0

    tx = dx / vx if vx > 0 else 0
    ty = dy / vy if vy > 0 else 0
    tz = dz / vz if vz > 0 else 0

    # 各軸同動，完成時間取最大值
    time_sec = max(tx, ty, tz)

    return time_sec
```

這個做法比「三維距離 / rapid_feed」更合理。

## Algorithm 12：圓弧長度計算，IJK 版本

G02/G03 圓弧要依照平面 G17/G18/G19 解讀。

```python
function compute_arc_length_ijk(block):

    plane = block.plane

    start = block.start
    end = block.end
    I, J, K = block.ijk

    # 1. 根據平面選擇圓弧座標
    if plane == "G17":        # XY plane
        s1, s2 = start.x, start.y
        e1, e2 = end.x, end.y
        c1 = start.x + I
        c2 = start.y + J

    elif plane == "G18":      # XZ plane
        s1, s2 = start.x, start.z
        e1, e2 = end.x, end.z
        c1 = start.x + I
        c2 = start.z + K

    elif plane == "G19":      # YZ plane
        s1, s2 = start.y, start.z
        e1, e2 = end.y, end.z
        c1 = start.y + J
        c2 = start.z + K

    else:
        block.warning.append("Unknown arc plane")
        return compute_line_length(start, end)

    # 2. 半徑
    r_start = sqrt((s1 - c1)^2 + (s2 - c2)^2)
    r_end   = sqrt((e1 - c1)^2 + (e2 - c2)^2)

    # 3. 檢查起點半徑與終點半徑是否一致
    if abs(r_start - r_end) > tolerance:
        block.warning.append("Arc radius mismatch")

    radius = (r_start + r_end) / 2

    # 4. 計算起點角度與終點角度
    theta_start = atan2(s2 - c2, s1 - c1)
    theta_end   = atan2(e2 - c2, e1 - c1)

    # 5. 根據 G02/G03 決定角度方向
    if block.direction == "G03":  # CCW
        delta_theta = theta_end - theta_start
        if delta_theta <= 0:
            delta_theta += 2*pi

    elif block.direction == "G02":  # CW
        delta_theta = theta_start - theta_end
        if delta_theta <= 0:
            delta_theta += 2*pi

    # 6. 圓弧長度
    arc_length = radius * delta_theta

    # 7. 若有非平面軸變化，代表螺旋插補，要加入第三軸長度
    linear_axis_delta = compute_out_of_plane_delta(block)

    if abs(linear_axis_delta) > 0:
        # 螺旋線長度
        arc_length = sqrt(arc_length^2 + linear_axis_delta^2)

    return arc_length
```

第一階段建議優先完整支援 IJK。R 圓弧可以做，但要注意大小圓弧歧義。

## Algorithm 13：圓弧長度計算，R 版本

```python
function compute_arc_length_r(block):

    start_2d, end_2d = project_to_active_plane(block.start, block.end, block.plane)

    chord = distance_2d(start_2d, end_2d)
    radius = abs(block.r)

    if chord > 2 * radius:
        block.warning.append("Invalid arc R: chord larger than diameter")
        return compute_line_length(block.start, block.end)

    # 1. 由弦長與半徑計算夾角
    angle = 2 * asin(chord / (2 * radius))

    # 2. R 正負通常影響大圓弧/小圓弧
    if block.r < 0:
        angle = 2*pi - angle

    # 3. 圓弧長度
    arc_length = radius * angle

    # 4. 處理螺旋插補
    linear_axis_delta = compute_out_of_plane_delta(block)

    if abs(linear_axis_delta) > 0:
        arc_length = sqrt(arc_length^2 + linear_axis_delta^2)

    return arc_length
```

注意：R 圓弧在不同控制器細節可能不同，第一階段要在報告中標註「R 圓弧為近似解析」。

# 十一、固定循環展開演算法

第一階段可先支援常見 G81/G82/G83。若時間不夠，可以先列為 Phase 1.5。

## Algorithm 14：G81 鑽孔循環展開

G81 大致動作：

```
快速移動到孔位 XY
快速下到 R 平面
進給切削到 Z 深度
快速退回 R 平面
```

```python
function expand_g81_cycle(tokens, state):

    blocks = []

    current = state.current_position

    target_x = tokens.get("X", current.x)
    target_y = tokens.get("Y", current.y)
    z_depth = tokens["Z"]
    r_plane = tokens["R"]

    # 1. XY 快速定位
    p1 = (target_x, target_y, current.z)
    blocks.append(RapidMoveBlock(start=current, end=p1))

    # 2. 快速到 R 平面
    p2 = (target_x, target_y, r_plane)
    blocks.append(RapidMoveBlock(start=p1, end=p2))

    # 3. 進給鑽到 Z 深度
    p3 = (target_x, target_y, z_depth)
    blocks.append(LinearMoveBlock(start=p2, end=p3, feedrate=state.feedrate))

    # 4. 快速退回 R 平面
    blocks.append(RapidMoveBlock(start=p3, end=p2))

    # 5. 更新位置
    state.current_position = p2

    return blocks
```

## Algorithm 15：G83 啄鑽循環展開

G83 是分段鑽孔，會多次下刀與退刀。

```python
function expand_g83_cycle(tokens, state, machine_profile):

    blocks = []

    current = state.current_position

    target_x = tokens.get("X", current.x)
    target_y = tokens.get("Y", current.y)
    z_final = tokens["Z"]
    r_plane = tokens["R"]
    q_depth = tokens["Q"]      # 每次啄鑽深度

    # 1. 快速定位到 XY
    p_xy = (target_x, target_y, current.z)
    blocks.append(RapidMoveBlock(start=current, end=p_xy))

    # 2. 快速到 R 平面
    p_r = (target_x, target_y, r_plane)
    blocks.append(RapidMoveBlock(start=p_xy, end=p_r))

    # 3. 分段鑽孔
    current_z = r_plane

    while current_z > z_final:

        next_z = max(current_z - q_depth, z_final)

        # 3.1 進給鑽到 next_z
        p_cut_start = (target_x, target_y, current_z)
        p_cut_end = (target_x, target_y, next_z)
        blocks.append(LinearMoveBlock(
            start=p_cut_start,
            end=p_cut_end,
            feedrate=state.feedrate
        ))

        # 3.2 快速退回 R 平面排屑
        p_retract = (target_x, target_y, r_plane)
        blocks.append(RapidMoveBlock(
            start=p_cut_end,
            end=p_retract
        ))

        # 3.3 快速回到接近前一深度的位置
        # 可設定 clearance，例如距離上次底部 1mm
        approach_z = next_z + machine_profile.cycle.peck_clearance_mm

        if approach_z < r_plane:
            blocks.append(RapidMoveBlock(
                start=p_retract,
                end=(target_x, target_y, approach_z)
            ))
            current_z = approach_z
        else:
            current_z = r_plane

        # 下一次從 approach_z 繼續鑽
        current_z = next_z

    # 4. 最後位置回到 R 平面
    state.current_position = p_r

    return blocks
```

G83 不同控制器差異較大，第一階段做近似即可，並在 GUI 顯示「固定循環為展開估算」。

```python
function estimate_block_time(block, machine_profile):

    elif block.type == "SpindleEvent":
```

# 十四、第一階段可加入的簡化加減速修正

雖然完整 S 曲線建議放到第二階段，但第一階段可以先加入「簡化梯形加減速」作為選配。這可以讓短線段估測不要太樂觀。

## Algorithm 19：簡化梯形速度曲線

```python
function compute_trapezoid_time(distance, target_velocity, max_acc):

    # distance: mm
    # target_velocity: mm/s
    # max_acc: mm/s^2

    if distance <= 0:
        return 0

    # 1. 加速到目標速度所需時間
    t_acc = target_velocity / max_acc

    # 2. 加速距離
    d_acc = 0.5 * max_acc * t_acc^2

    # 3. 對稱減速，所以總加減速距離
    d_acc_dec = 2 * d_acc

    # 4. 如果距離夠長，可以達到目標速度
    if distance >= d_acc_dec:

        d_const = distance - d_acc_dec
        t_const = d_const / target_velocity

        total_time = t_acc + t_const + t_acc

    # 5. 如果距離太短，無法達到目標速度
    else:

        # 三角速度曲線
        # distance = 2 * (0.5 * a * t^2) = a * t^2
        t_peak = sqrt(distance / max_acc)

        total_time = 2 * t_peak

    return total_time
```

第一階段可以在 Machine Profile 裡提供模式：

```yaml
time_model:
  mode: "constant_velocity"   # 或 "trapezoid"
```

然後在估算 G01/G02/G03 時切換：

```python
function compute_feed_move_time_with_model(length_mm, feed_mm_min, machine_profile):

    v = feed_mm_min / 60.0

    if machine_profile.time_model.mode == "constant_velocity":
        return length_mm / v

    elif machine_profile.time_model.mode == "trapezoid":
        a = machine_profile.default_cut_acc_mm_s2

```

注意：這仍然不是真正控制器 Look-ahead，只是讓第一階段比純定速更不樂觀。

```python
function summarize_result(ir_program):

    result = ResultModel()

    result.total_time_sec = 0
    result.rapid_time_sec = 0
    result.cutting_time_sec = 0
    result.arc_time_sec = 0
    result.dwell_time_sec = 0
    result.tool_change_time_sec = 0
    result.spindle_time_sec = 0
    result.coolant_time_sec = 0
    result.warning_list = []

    for block in ir_program:

        result.total_time_sec += block.estimated_time

        if block.type == "RapidMove":
            result.rapid_time_sec += block.estimated_time

        elif block.type == "LinearMove":
            result.cutting_time_sec += block.estimated_time

        elif block.type == "ArcMove":
            result.arc_time_sec += block.estimated_time
            result.cutting_time_sec += block.estimated_time

        elif block.type == "Dwell":
            result.dwell_time_sec += block.estimated_time

        elif block.type == "ToolChange":
            result.tool_change_time_sec += block.estimated_time
            result.tool_change_count += 1

        elif block.type == "SpindleEvent":
            result.spindle_time_sec += block.estimated_time

        elif block.type == "CoolantEvent":
            result.coolant_time_sec += block.estimated_time

        if block.warning not empty:
            result.warning_list.extend(block.warning)

    # 轉成 HH:MM:SS
    result.total_time_text = format_seconds(result.total_time_sec)

    # 產生 GUI 表格資料
    result.block_table = build_block_table(ir_program)

    return result
```
