以下針對第二階段「數位雙生型加工時間估測模組」設計演算法與虛擬碼。第二階段是沿用第一階段產生的 IR，再加入「機台限制、轉角減速、Look-ahead、S 曲線加減速與校正」。第二階段的核心應轉向模擬 CNC 控制器的運動規劃行為。

要先說明一點：不同 CNC 控制器，例如 FANUC、Siemens、Heidenhain 的實際 Look-ahead 與平滑控制演算法通常不是完全公開。因此第二階段建議做「工程近似模型」，再透過實測校正讓模型貼近指定機台，而不是假裝可以 100% 還原控制器內部邏輯。

---

# 一、第二階段核心概念

第一階段的時間模型是：

```
時間 = 路徑長度 / 指令進給率
```

第二階段的時間模型改成：

```
時間 = 虛擬控制器根據機台限制實際跑完該段路徑所需時間
```

也就是每段路徑都要考慮：

```
1. 該段長度 s
2. 該段方向 vector
3. 指令進給率 F
4. 機台最大速度限制
5. 機台最大加速度限制
6. 機台最大 Jerk 限制
7. 前一段進入速度
8. 下一段轉角允許速度
9. 該段是否短到無法達到指令速度
```

第二階段整體流程如下：

```
第一階段 IR Program
   ↓
Motion Segment 化
   ↓
機台運動學映射
   ↓
計算每段速度 / 加速度 / Jerk 上限
   ↓
計算轉角 Junction Velocity
   ↓
Look-ahead Backward Pass
   ↓
Look-ahead Forward Pass
   ↓
七段式 S 曲線時間計算
   ↓
短線段退化處理
   ↓
輸出總時間、分段時間、速度曲線、瓶頸區段
```

# 三、第二階段主流程演算法

## Algorithm 1：第二階段總流程

```python
function estimate_time_phase2(ir_program, machine_profile):

    # 1. 從第一階段 IR 取出真正會產生運動的 block
    motion_blocks = filter_motion_blocks(ir_program)

    # 2. 將 Arc 可選擇保留為 arc，或離散成小線段
    # 第二階段建議先離散化，方便統一做 Look-ahead
    segments = build_motion_segments(motion_blocks, machine_profile)

    # 3. 機台運動學映射
    # 三軸：工件座標 ≈ 機台座標
    # 五軸：工件座標 + 刀軸向量 → X/Y/Z/A/B/C
    for segment in segments:
        apply_kinematic_mapping(segment, machine_profile)

    # 4. 計算每段的速度 / 加速度 / Jerk 上限
    for segment in segments:
        compute_segment_limits(segment, machine_profile)

    # 5. 計算每個交界點的轉角速度限制
    junctions = compute_all_junction_limits(segments, machine_profile)

    # 6. 初始化節點速度
    node_velocities = initialize_node_velocities(segments, junctions)

    # 7. Look-ahead 雙向掃描
    node_velocities = backward_pass(
        segments,
        node_velocities,
        machine_profile
    )

    node_velocities = forward_pass(
        segments,
        node_velocities,
        machine_profile
    )

    # 8. 將節點速度寫回 segment
    assign_entry_exit_velocity(segments, node_velocities)

    # 9. 每段套用七段式 S 曲線 / 退化 S 曲線
    for segment in segments:
        profile = compute_s_curve_segment_profile(segment)
        segment.estimated_time = profile.total_time
        segment.peak_velocity = profile.peak_velocity
        segment.dynamic_profile = profile.samples

    # 10. 合併非運動事件時間，例如 M06、G04、M03
    event_time = compute_event_time(ir_program, machine_profile)

    # 11. 統計結果
    result = summarize_phase2_result(
        segments=segments,
        event_time=event_time,
        junctions=junctions
    )

    return result
```

這個主流程中，真正的核心是第 5 到第 9 步：轉角速度、雙向掃描、S 曲線。

# 四、Motion Segment 建立演算法

第一階段 IR 可能包含 `RapidMoveBlock`、`LinearMoveBlock`、`ArcMoveBlock`。第二階段要轉成統一的 `MotionSegment`。

## Algorithm 2：建立 Motion Segment

```python
function build_motion_segments(motion_blocks, machine_profile):

    segments = []

    for block in motion_blocks:

        if block.type == "RapidMove":
            segment = create_linear_segment_from_block(block)
            segment.motion_type = "rapid"
            segment.commanded_feed = machine_profile.rapid_feed_mm_s
            segments.append(segment)

        elif block.type == "LinearMove":
            segment = create_linear_segment_from_block(block)
            segment.motion_type = "linear"
            segment.commanded_feed = resolve_feedrate_mm_s(block)
            segments.append(segment)

        elif block.type == "ArcMove":

            # 第二階段建議先將圓弧離散化成小線段
            arc_segments = discretize_arc_block(
                block=block,
                chord_tolerance=machine_profile.arc_chord_tolerance_mm
            )

            for seg in arc_segments:
                seg.motion_type = "arc_discretized"
                seg.commanded_feed = resolve_feedrate_mm_s(block)
                segments.append(seg)

    # 移除長度太小的 segment
    segments = remove_zero_length_segments(segments)

    return segments
```

# 五、圓弧離散化演算法

為什麼第二階段建議把圓弧離散化？因為 Look-ahead、轉角速度、S 曲線規劃比較容易統一處理。後續若要更精準，可以保留 arc primitive。

## Algorithm 3：圓弧離散化

```python
function discretize_arc_block(block, chord_tolerance):

    center = compute_arc_center(block)
    radius = compute_arc_radius(block)
    start_angle = compute_start_angle(block, center)
    end_angle = compute_end_angle(block, center)
    delta_angle = compute_sweep_angle(block.direction, start_angle, end_angle)

    # 根據 chord tolerance 決定每段角度
    # chord_error ≈ r * (1 - cos(delta_theta / 2))
    max_delta_theta = 2 * arccos(1 - chord_tolerance / radius)

    segment_count = ceil(abs(delta_angle) / max_delta_theta)

    arc_segments = []

    previous_point = block.start

    for i in range(1, segment_count + 1):

        ratio = i / segment_count
        theta = start_angle + ratio * delta_angle

        next_point = point_on_arc(center, radius, theta, block.plane)

        # 若是螺旋插補，同步內插非平面軸
        next_point = interpolate_helical_axis(
            block.start,
            block.end,
            next_point,
            ratio,
            block.plane
        )

        seg = MotionSegment(
            block_id=block.id,
            line_no=block.line_no,
            raw=block.raw,
            start_pos=previous_point,
            end_pos=next_point
        )

        seg.length = distance(previous_point, next_point)
        seg.direction = normalize(next_point - previous_point)

        arc_segments.append(seg)

        previous_point = next_point

    return arc_segments
```

這裡的 `chord_tolerance` 不宜太小。太小會造成 segment 過多，計算量暴增。第一版可以設定 0.01 mm 或 0.02 mm，視機台精度需求調整。

# 六、機台運動學映射演算法

第二階段至少要支援三軸，並預留五軸介面。

三軸情況下：

```
工件座標 X/Y/Z = 機台軸 X/Y/Z
```

五軸情況下：

```
刀尖點位置 + 刀軸方向 → 逆向運動學 → X/Y/Z/A/B/C
```

## Algorithm 4：機台運動學映射主函式

```python
function apply_kinematic_mapping(segment, machine_profile):

    if machine_profile.kinematic_type == "3_axis":

        segment.axis_start = {
            "X": segment.start_pos.x,
            "Y": segment.start_pos.y,
            "Z": segment.start_pos.z
        }

        segment.axis_end = {
            "X": segment.end_pos.x,
            "Y": segment.end_pos.y,
            "Z": segment.end_pos.z
        }

    elif machine_profile.kinematic_type == "5_axis_head_table":

        segment.axis_start = inverse_kinematics_5axis(
            tcp_position=segment.start_pos,
            tool_axis=segment.start_tool_axis,
            machine_profile=machine_profile
        )

        segment.axis_end = inverse_kinematics_5axis(
            tcp_position=segment.end_pos,
            tool_axis=segment.end_tool_axis,
            machine_profile=machine_profile
        )

    else:
        raise Error("Unsupported kinematic type")

    segment.axis_delta = {}

    for axis in segment.axis_start.keys():
        segment.axis_delta[axis] = segment.axis_end[axis] - segment.axis_start[axis]
```

## Algorithm 5：五軸逆向運動學介面

第一版可以先保留介面，不一定完整實作。

```python
function inverse_kinematics_5axis(tcp_position, tool_axis, machine_profile):

    # 1. 讀取機台結構參數
    #    例如旋轉軸順序、pivot length、A/C 軸限制

    kinematic_chain = machine_profile.kinematic_chain

    # 2. 根據 tool_axis 解算旋轉軸角度
    #    例如求 A, C 或 B, C

    rotary_solution = solve_rotary_axes(
        tool_axis,
        kinematic_chain
    )

    # 3. 根據旋轉軸造成的 TCP 偏移，反算線性軸位置

    linear_solution = solve_linear_axes(
        tcp_position,
        rotary_solution,
        machine_profile
    )

    # 4. 檢查軸限制
    check_axis_limits(linear_solution, rotary_solution)

    return merge(linear_solution, rotary_solution)
```

第二階段若目標先是三軸或車銑簡化模型，五軸可以暫時只做介面。不要在第二階段一開始就把五軸逆向運動學與 Look-ahead 全部一起做，風險很高。

# 七、軸向限制轉換成路徑限制

這是第二階段非常重要的部分。

NC-Code 指令的是刀尖路徑速度，但真實限制來自每個機台軸。

假設某段路徑方向在機台軸空間的單位向量是：

```
u = [ux, uy, uz]
```

則路徑速度 v 對每個軸造成的速度是：

```
vx = v * ux
vy = v * uy
vz = v * uz
```

所以路徑最大速度要受各軸限制：

```
v_path_max = min(axis_vmax / abs(ui))
```

加速度與 Jerk 也類似。

## Algorithm 6：計算每段速度、加速度、Jerk 上限

```python
function compute_segment_limits(segment, machine_profile):

    # 1. 取得軸向位移
    axis_delta = segment.axis_delta

    # 2. 計算軸空間長度
    axis_length = sqrt(sum(delta^2 for delta in axis_delta.values()))

    if axis_length <= epsilon:
        segment.v_cap = 0
        segment.a_cap = 0
        segment.j_cap = 0
        segment.warnings.append("Zero axis movement")
        return

    # 3. 軸空間方向單位向量
    axis_unit = {}

    for axis, delta in axis_delta.items():
        axis_unit[axis] = delta / axis_length

    # 4. 計算該段路徑速度上限
    v_limits = []
    a_limits = []
    j_limits = []

    for axis, u in axis_unit.items():

        if abs(u) < epsilon:
            continue

        axis_limit = machine_profile.axes[axis]

        v_limits.append(axis_limit.max_velocity / abs(u))
        a_limits.append(axis_limit.max_acc / abs(u))
        j_limits.append(axis_limit.max_jerk / abs(u))

    axis_v_cap = min(v_limits)
    axis_a_cap = min(a_limits)
    axis_j_cap = min(j_limits)

    # 5. 指令進給率也要納入限制
    segment.v_cap = min(
        segment.commanded_feed,
        axis_v_cap,
        machine_profile.max_cut_feed_mm_s
    )

    segment.a_cap = min(
        axis_a_cap,
        machine_profile.default_cut_acc_mm_s2
    )

    segment.j_cap = min(
        axis_j_cap,
        machine_profile.default_cut_jerk_mm_s3
    )

    # 6. G00 可以套用 rapid 限制
    if segment.motion_type == "rapid":
        segment.v_cap = min(segment.v_cap, machine_profile.rapid_feed_mm_s)
```

這個演算法能讓估測從「幾何路徑」升級成「機台軸限制」。例如同一段 TCP 路徑，在五軸機台上可能造成旋轉軸大幅運動，因此實際時間會被旋轉軸拖慢。

# 八、轉角 Junction Velocity 演算法

兩段路徑方向不同時，機台不能無限制高速通過交界點。轉角越尖、容許誤差越小，通過速度越低。

## Algorithm 7：計算單一 Junction Velocity

```python
function compute_junction_velocity(prev_segment, next_segment, machine_profile):

    d1 = prev_segment.direction
    d2 = next_segment.direction

    # 1. 若其中一段長度太小，要求停下
    if prev_segment.length < epsilon or next_segment.length < epsilon:
        return 0.0, "zero length segment"

    # 2. 計算方向夾角
    dot_value = clamp(dot(d1, d2), -1.0, 1.0)
    phi = arccos(dot_value)      # 0 = 同方向, pi = 反方向

    # 3. 幾乎同方向，不需要轉角減速
    if phi < machine_profile.controller.same_direction_angle_threshold:
        return min(prev_segment.v_cap, next_segment.v_cap), "almost straight"

    # 4. 幾乎反方向，必須接近停止
    if abs(pi - phi) < machine_profile.controller.reverse_angle_threshold:
        return 0.0, "reverse direction"

    # 5. 取兩段較保守的加速度
    a = min(prev_segment.a_cap, next_segment.a_cap)

    # 6. 轉角容許誤差
    epsilon_corner = machine_profile.junction_tolerance_mm

    # 7. 工程近似的虛擬圓弧半徑
    # 使用 controller-style junction deviation 近似
    alpha = arccos(clamp(-dot_value, -1.0, 1.0))
    sin_half = sin(alpha / 2)

    if sin_half <= small_value:
        return 0.0, "sharp corner"

    r_blend = epsilon_corner * sin_half / (1 - sin_half)

    # 8. 向心加速度限制：v^2 / r <= a
    v_acc_limit = sqrt(a * r_blend)

    # 9. 可選：加入 jerk 限制的保守限制
    j = min(prev_segment.j_cap, next_segment.j_cap)
    v_jerk_limit = cubic_root(j * r_blend * r_blend)

    # 10. 取最小值
    v_junction = min(
        prev_segment.v_cap,
        next_segment.v_cap,
        v_acc_limit,
        v_jerk_limit
    )

    return v_junction, "corner limit"
```

這個 Junction Velocity 是近似模型，不是某一家控制器的精確內部公式。實務上會再靠機台校正調整 `junction_tolerance_mm`、`a_cap`、`j_cap`。

## Algorithm 8：計算所有 Junction

```python
function compute_all_junction_limits(segments, machine_profile):

    junctions = []

    # 節點數量 = segment 數量 + 1
    # node i 是 segment i-1 和 segment i 的交界

    for i in range(1, len(segments)):

        prev_seg = segments[i - 1]
        next_seg = segments[i]

        # 若前後是不同 motion group，例如切削後換刀，強制停止
        if should_force_stop_between(prev_seg, next_seg):
            v_limit = 0.0
            reason = "force stop between motion groups"

        else:
            v_limit, reason = compute_junction_velocity(
                prev_seg,
                next_seg,
                machine_profile
            )

        junction = Junction(
            index=i,
            prev_segment_id=prev_seg.block_id,
            next_segment_id=next_seg.block_id,
            angle_rad=angle_between(prev_seg.direction, next_seg.direction),
            tolerance=machine_profile.junction_tolerance_mm,
            v_junction_limit=v_limit,
            reason=reason
        )

        junctions.append(junction)

    return junctions
```

# 九、初始化節點速度

如果有 N 段 segment，就有 N+1 個節點速度。

```
segment 0: node[0] → node[1]
segment 1: node[1] → node[2]
segment 2: node[2] → node[3]
```

每個 node 速度受到 junction velocity 限制。

## Algorithm 9：初始化 Node Velocity

```python
function initialize_node_velocities(segments, junctions):

    n = len(segments)

    node_velocities = array(size=n + 1)

    # 起點通常從 0 開始
    node_velocities[0] = 0.0

    # 終點通常要停下
    node_velocities[n] = 0.0

    # 中間節點用 junction 限制
    for junction in junctions:
        node_velocities[junction.index] = junction.v_junction_limit

    return node_velocities
```

如果某些連續路徑允許不停頓，node velocity 就會是 junction velocity；如果換刀、M00、M01、主軸停止，則應強制 node velocity = 0。

# 十、S 曲線基礎函式

雙向掃描需要知道一件事：

```
在距離 s 內，能不能從速度 v0 加速 / 減速到速度 v1？
```

所以要先建立 S 曲線基礎函式。

## Algorithm 10：計算速度轉換所需最短時間與距離

這個函式只計算從 `v0` 變到 `v1` 的 S 曲線，不含等速段。

```python
function s_curve_transition(v0, v1, a_max, j_max):

    dv = abs(v1 - v0)

    if dv <= epsilon:
        return TransitionResult(
            time=0,
            distance=0,
            profile_type="no_change"
        )

    # 1. 判斷是否能達到最大加速度
    # 若 dv 太小，連最大加速度都達不到，屬於 triangular jerk profile
    dv_threshold = a_max * a_max / j_max

    if dv <= dv_threshold:

        # Triangular S-curve
        # 加速度從 0 上升後又回到 0，中間沒有等加速度段
        t_j = sqrt(dv / j_max)
        total_time = 2 * t_j

        # 近似距離：平均速度 × 時間
        distance = ((v0 + v1) / 2) * total_time

        return TransitionResult(
            time=total_time,
            distance=distance,
            t_j=t_j,
            t_a=0,
            profile_type="triangular_s_curve"
        )

    else:

        # Full S-curve acceleration/deceleration
        t_j = a_max / j_max

        # 剩餘速度變化由等加速度段完成
        t_a = (dv - dv_threshold) / a_max

        total_time = 2 * t_j + t_a

        distance = ((v0 + v1) / 2) * total_time

        return TransitionResult(
            time=total_time,
            distance=distance,
            t_j=t_j,
            t_a=t_a,
            profile_type="full_s_curve"
        )
```

這是工程上可用的版本。若後續要做到更精準，可以把每個 jerk phase 的位移積分完整展開。

# 十一、可達速度計算

Look-ahead 的 Backward Pass / Forward Pass 都需要計算「在有限距離內，最多能把速度提高或降低到多少」。

## Algorithm 11：給定終點速度，反推起點最高速度

用於 Backward Pass。

```python
function max_start_velocity_given_end_velocity(
    distance,
    v_end,
    v_cap,
    a_max,
    j_max
):

    low = v_end
    high = v_cap

    for iteration in range(MAX_ITER):

        mid = (low + high) / 2

        transition = s_curve_transition(
            v0=mid,
            v1=v_end,
            a_max=a_max,
            j_max=j_max
        )

        if transition.distance <= distance:
            # 距離足夠從 mid 減速到 v_end
            low = mid
        else:
            # mid 太快，距離不夠減速
            high = mid

    return low
```

## Algorithm 12：給定起點速度，推算終點最高速度

用於 Forward Pass。

```python
function max_end_velocity_given_start_velocity(
    distance,
    v_start,
    v_cap,
    a_max,
    j_max
):

    low = v_start
    high = v_cap

    for iteration in range(MAX_ITER):

        mid = (low + high) / 2

        transition = s_curve_transition(
            v0=v_start,
            v1=mid,
            a_max=a_max,
            j_max=j_max
        )

        if transition.distance <= distance:
            # 距離足夠從 v_start 加速到 mid
            low = mid
        else:
            # mid 太快，距離不夠加速
            high = mid

    return low
```

# 十二、Look-ahead Backward Pass

Backward Pass 的目的：

```
從後往前看，確保遇到低速轉角前，前面的速度可以來得及降下來。
```

## Algorithm 13：Backward Pass

```python
function backward_pass(segments, node_velocities, machine_profile):

    n = len(segments)

    # 從最後一段往前掃
    for i in reversed(range(0, n)):

        seg = segments[i]

        # segment i: node i → node i+1
        v_end = node_velocities[i + 1]

        # 在該段長度內，可以從多高的起點速度減速到 v_end？
        max_v_start = max_start_velocity_given_end_velocity(
            distance=seg.length,
            v_end=v_end,
            v_cap=seg.v_cap,
            a_max=seg.a_cap,
            j_max=seg.j_cap
        )

        # node i 不能超過此速度
        node_velocities[i] = min(
            node_velocities[i],
            max_v_start,
            seg.v_cap
        )

    return node_velocities
```

如果下一個轉角很尖，`node[i+1]` 會很低，Backward Pass 會把前面多段速度逐步壓低，模擬控制器提前減速。

# 十三、Look-ahead Forward Pass

Forward Pass 的目的：

```
從前往後看，確保每段速度提升不超過機台加速能力。
```

## Algorithm 14：Forward Pass

```python
function forward_pass(segments, node_velocities, machine_profile):

    n = len(segments)

    # 從第一段往後掃
    for i in range(0, n):

        seg = segments[i]

        # segment i: node i → node i+1
        v_start = node_velocities[i]

        # 在該段長度內，最多能加速到多高的終點速度？
        max_v_end = max_end_velocity_given_start_velocity(
            distance=seg.length,
            v_start=v_start,
            v_cap=seg.v_cap,
            a_max=seg.a_cap,
            j_max=seg.j_cap
        )

        # node i+1 不能超過此速度
        node_velocities[i + 1] = min(
            node_velocities[i + 1],
            max_v_end,
            seg.v_cap
        )

    return node_velocities
```

在實作上可以跑一次 Backward + Forward。若想更穩，可以跑到收斂：

```python
function bidirectional_lookahead(segments, node_velocities, machine_profile):

    for iteration in range(MAX_LOOKAHEAD_ITER):

        old_velocities = copy(node_velocities)

        node_velocities = backward_pass(
            segments,
            node_velocities,
            machine_profile
        )

        node_velocities = forward_pass(
            segments,
            node_velocities,
            machine_profile
        )

        if max_abs(node_velocities - old_velocities) < velocity_tolerance:
            break

    return node_velocities
```

# 十五、七段式 S 曲線時間計算

每一段 segment 的問題是：

```
已知：
s      = 路徑長度
v0     = 起點速度
v1     = 終點速度
v_cap  = 最高速度
a_max  = 最大加速度
j_max  = 最大 Jerk

求：
該段實際時間
```

## Algorithm 16：S 曲線 Segment 時間計算

```python
function compute_s_curve_segment_profile(segment):

    s = segment.length
    v0 = segment.entry_velocity
    v1 = segment.exit_velocity
    v_cap = segment.v_cap
    a = segment.a_cap
    j = segment.j_cap

    # 1. 嘗試加速到 v_cap，再從 v_cap 減速到 v1
    acc = s_curve_transition(
        v0=v0,
        v1=v_cap,
        a_max=a,
        j_max=j
    )

    dec = s_curve_transition(
        v0=v_cap,
        v1=v1,
        a_max=a,
        j_max=j
    )

    distance_need = acc.distance + dec.distance

    # 2. 如果距離足夠，可以達到 v_cap，中間有等速段
    if distance_need <= s:

        cruise_distance = s - distance_need
        cruise_time = cruise_distance / v_cap if v_cap > 0 else 0

        total_time = acc.time + cruise_time + dec.time

        profile = build_dynamic_profile(
            v0=v0,
            v_peak=v_cap,
            v1=v1,
            acc_transition=acc,
            dec_transition=dec,
            cruise_time=cruise_time,
            segment=segment
        )

        return SegmentProfile(
            total_time=total_time,
            peak_velocity=v_cap,
            profile_type="full_s_curve_with_cruise",
            samples=profile
        )

    # 3. 如果距離不夠，代表短線段退化，無法達到 v_cap
    else:

        v_peak = solve_reachable_peak_velocity(
            distance=s,
            v0=v0,
            v1=v1,
            v_cap=v_cap,
            a_max=a,
            j_max=j
        )

        acc = s_curve_transition(
            v0=v0,
            v1=v_peak,
            a_max=a,
            j_max=j
        )

        dec = s_curve_transition(
            v0=v_peak,
            v1=v1,
            a_max=a,
            j_max=j
        )

        total_time = acc.time + dec.time

        profile = build_dynamic_profile(
            v0=v0,
            v_peak=v_peak,
            v1=v1,
            acc_transition=acc,
            dec_transition=dec,
            cruise_time=0,
            segment=segment
        )

        return SegmentProfile(
            total_time=total_time,
            peak_velocity=v_peak,
            profile_type="degraded_s_curve_no_cruise",
            samples=profile
        )
```

# 十六、短線段退化求解演算法

短線段退化的核心是求出該段真正能達到的最高速度 `v_peak`。

## Algorithm 17：求短線段可達峰值速度

```python
function solve_reachable_peak_velocity(
    distance,
    v0,
    v1,
    v_cap,
    a_max,
    j_max
):

    low = max(v0, v1)
    high = v_cap

    for iteration in range(MAX_ITER):

        mid = (low + high) / 2

        acc = s_curve_transition(
            v0=v0,
            v1=mid,
            a_max=a_max,
            j_max=j_max
        )

        dec = s_curve_transition(
            v0=mid,
            v1=v1,
            a_max=a_max,
            j_max=j_max
        )

        required_distance = acc.distance + dec.distance

        if required_distance <= distance:
            # 可以達到 mid，嘗試更高
            low = mid
        else:
            # 距離不夠，降低峰值速度
            high = mid

    return low
```

這個演算法可以處理大量微小線段。它會自然產生一種結果：短線段即使 F 值很高，實際速度也上不去。

# 十七、動態曲線取樣演算法

第二階段不只輸出時間，也應輸出速度、加速度、Jerk 曲線。GUI 可以用這些資料畫圖。

## Algorithm 18：建立動態 Profile

```python
function build_dynamic_profile(
    v0,
    v_peak,
    v1,
    acc_transition,
    dec_transition,
    cruise_time,
    segment
):

    samples = []

    dt = machine_profile.interpolation_period_s

    t = 0
    s_accum = 0

    # 1. 建立加速段 jerk phases
    acc_phases = build_scurve_phases(
        v_start=v0,
        v_end=v_peak,
        transition=acc_transition,
        direction="acc"
    )

    # 2. 等速段
    cruise_phase = Phase(
        duration=cruise_time,
        jerk=0,
        acceleration=0,
        velocity=v_peak
    )

    # 3. 建立減速段 jerk phases
    dec_phases = build_scurve_phases(
        v_start=v_peak,
        v_end=v1,
        transition=dec_transition,
        direction="dec"
    )

    all_phases = acc_phases + [cruise_phase] + dec_phases

    current_v = v0
    current_a = 0

    for phase in all_phases:

        phase_elapsed = 0

        while phase_elapsed < phase.duration:

            # 3.1 根據 jerk 更新 acceleration
            current_a = current_a + phase.jerk * dt

            # 3.2 根據 acceleration 更新 velocity
            current_v = current_v + current_a * dt

            # 3.3 根據 velocity 更新 displacement
            s_accum = s_accum + current_v * dt

            # 3.4 儲存取樣資料
            samples.append({
                "time": t,
                "position_s": s_accum,
                "velocity": current_v,
                "acceleration": current_a,
                "jerk": phase.jerk,
                "block_id": segment.block_id,
                "line_no": segment.line_no
            })

            t = t + dt
            phase_elapsed = phase_elapsed + dt

    return samples
```

這裡是數值取樣版本，容易實作。未來若要提高精度，可以改用解析式積分。

# 十八、瓶頸區段偵測演算法

第二階段很有價值的一點是：不只告訴你總時間，還可以告訴你哪裡拖慢。

## Algorithm 19：偵測慢段與瓶頸原因

```python
function detect_bottlenecks(segments, junctions):

    bottlenecks = []

    for seg in segments:

        ideal_time = seg.length / seg.commanded_feed
        actual_time = seg.estimated_time

        if ideal_time <= epsilon:
            continue

        slowdown_ratio = actual_time / ideal_time

        reason = []

        # 1. 短線段導致速度上不去
        if seg.peak_velocity < 0.8 * seg.commanded_feed:
            reason.append("short segment speed degradation")

        # 2. 受機台軸速度限制
        if seg.v_cap < seg.commanded_feed:
            reason.append("axis velocity limit")

        # 3. 受加速度限制
        if actual_time > ideal_time * 1.2:
            reason.append("acceleration or jerk limited")

        # 4. 進出口速度偏低，通常是轉角造成
        if seg.entry_velocity < 0.5 * seg.v_cap:
            reason.append("low entry velocity due to previous junction")

        if seg.exit_velocity < 0.5 * seg.v_cap:
            reason.append("low exit velocity due to next junction")

        if slowdown_ratio > 1.2:
            bottlenecks.append({
                "line_no": seg.line_no,
                "raw": seg.raw,
                "ideal_time": ideal_time,
                "actual_time": actual_time,
                "slowdown_ratio": slowdown_ratio,
                "reason": reason
            })

    return bottlenecks
```

GUI 可以把這些區段標成「慢段熱點」。

# 二十四、最核心結論

第二階段的本質是把加工時間估測從：

```
幾何加總模型
```

升級成：

```
受機台物理限制的速度規劃模型
```

最核心的五個演算法是：

```
1. Axis-based limit mapping
   把機台各軸限制轉成該路徑段的速度 / 加速度 / Jerk 上限。

2. Junction Velocity
   根據轉角與容許誤差，限制轉角通過速度。

3. Bidirectional Look-ahead
   用 Backward Pass 提前減速，用 Forward Pass 限制加速。

4. S-curve Profile
   用 Jerk-limited 七段式加減速計算實際時間。

5. Calibration
   用實測加工時間反推機台參數，讓虛擬控制器貼近真實機台。
```
