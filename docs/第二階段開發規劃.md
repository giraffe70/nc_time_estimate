第二階段：數位雙生型加工時間估測模組(機台運動學數位雙生)

第二階段加入 S 曲線加減速、短線段退化、轉角減速、Look-ahead 與機台校正。目標是把第一階段從「幾何估算器」升級成「虛擬控制器」。這一階段要處理傳統估測最容易失準的地方：短線段、轉角減速、加速度限制、Jerk 限制與 Look-ahead。這一版才是真正能逼近實際 CNC 控制器行為的高精度加工時間估測模組。

若要把誤差從傳統的 10%–40% 降低，需要捨棄單純定速假設，導入 S 曲線加減速、短路徑退化與前瞻控制。

第二階段新增功能如下：

1. 建立完整 Machine Profile。
2. 實作七段式 S 曲線加減速。
3. 實作短線段速度退化。
4. 實作轉角 Junction Velocity 限制。
5. 實作 Look-ahead 前瞻緩衝區。
6. 支援雙向掃描速度規劃。
7. 支援機台校正流程。
8. 支援實測資料比對與誤差分析。
9. 輸出速度、加速度、Jerk 時序曲線。
10. 可標記造成加工時間變長的瓶頸區段。

第二階段核心流程如下：

```
NC-Code
   ↓
Parser
   ↓
Modal State Machine
   ↓
Toolpath IR
   ↓
計算每段幾何長度與方向
   ↓
計算每個轉角的允許通過速度
   ↓
Backward Pass：由後往前限制減速能力
   ↓
Forward Pass：由前往後限制加速能力
   ↓
每段套用 S 曲線或退化 S 曲線
   ↓
加總每段真實運動時間
   ↓
輸出總時間與動態曲線
```

這裡的重點是：每一段 NC-Code 不一定能跑到程式指定的 F 值。特別是大量小線段加工時，機台可能還沒加速到指定速度就要減速，因此真實時間會比「距離 ÷ F」長很多。

## 第二階段運動模擬細節

第二階段要把每一段路徑看成一個受物理限制的運動問題。

每一段運動有幾個條件：

```
路徑長度 s
起始速度 v0
目標速度 vf
段內最高速度 vmax
最大加速度 amax
最大 Jerk jmax
終點速度 v1
```

真實控制器不會每段都從 0 開始，也不一定每段都停下來。它會根據後面路徑決定這一段終點速度，也就是 Look-ahead 的作用。

第二階段的速度規劃可以這樣做：

```
1. 每一段先取得 NC 指定進給率。
2. 根據機台最大速度限制，得到該段最大可達速度。
3. 計算相鄰兩段的夾角。
4. 根據夾角與路徑容許誤差，計算轉角最大通過速度。
5. 用 backward pass 確保遇到急轉角前能來得及減速。
6. 用 forward pass 確保加速過程不超過機台能力。
7. 每段使用 S 曲線或退化 S 曲線計算實際時間。
```

轉角速度限制的本質是：角度越尖、容許誤差越小、機台加速度越低，過彎速度就必須越慢。文件中也提到，前瞻控制會透過雙向掃描，求解符合動力限制的速度分佈。
