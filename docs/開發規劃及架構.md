數位雙生模組應包含 NC 解析、IR 中介模型、機台運動學、S 曲線加減速與前瞻控制等層次。

## 開發目標

這套模組的本質不是「讀取 NC-Code 後加總路徑長度」，而是建立一個「虛擬 CNC 控制器」。它要模擬控制器如何讀 NC-Code、如何產生路徑、如何受限於機台速度、加速度、Jerk、轉角減速與快速移動限制，最後輸出接近實際機台的加工時間。

建議開發目標如下：

第一階段先完成可用版本：輸入 NC-Code，解析 G00、G01、G02、G03、G04、M-code、換刀、主軸、進給率、單位、絕對/相對座標，輸出「總加工時間、各段時間、空切/實切時間、警告訊息、路徑預覽」。

第二階段再升級成數位雙生版本：加入機台 Profile、S 曲線加減速、短線段退化、轉角速度限制、Look-ahead 前瞻、機台校正、誤差驗證，讓估測結果更貼近真實控制器行為。

第一階段做「看得懂 NC-Code 的時間估算器」，第二階段做「會模擬機台怎麼跑 NC-Code 的虛擬控制器」。

## 整體系統架構

建議系統架構如下：

```
NC-Code 檔案
   ↓
[1] NC-Code Parser 解析器
   ↓
[2] Modal State Machine 模態狀態機
   ↓
[3] Toolpath IR 中介表示層
   ↓
[4] Geometry Processor 幾何路徑處理
   ↓
[5] Motion Simulation Engine 運動模擬引擎
   ↓
[6] Machine Profile 機台特徵設定檔
   ↓
[7] Time Estimator 加工時間估測器
   ↓
[8] GUI / Report / Chart 結果呈現
```

各層的功能如下。

NC-Code Parser 負責把原始文字轉成結構化 Token，例如 `G01 X10.0 Y20.0 F800` 會被轉成 motion=G01、X=10、Y=20、F=800。

Modal State Machine 負責記住目前有效狀態。CNC 程式不是每一行都完整寫出所有參數，例如前一行是 `G01 F800`，下一行只寫 `X100`，那下一行仍然代表 G01，且進給率仍然是 F800。因此一定要做模態追蹤。

Toolpath IR 是中介資料格式。不要直接用原始 G-code 做時間計算，而是先轉成統一格式，例如 LineMove、ArcMove、RapidMove、Dwell、ToolChange、SpindleEvent。文件中也建議以 MotionBlock、ArcBlock、SpindleBlock 等 IR 物件作為後續前瞻控制與運動模擬的基礎。

Geometry Processor 負責計算每段路徑長度。G01 用直線距離，G02/G03 用圓弧長度，G81/G83 等固定循環則展開成多個細部動作。

Motion Simulation Engine 是第二階段核心。第一階段可以先用簡化模型；第二階段要加入加減速、S 曲線、Look-ahead 與轉角限制。

Machine Profile 用來描述機台能力，例如各軸最大速度、最大加速度、最大 Jerk、快速進給速度、插補週期、Look-ahead block 數量、轉角容許誤差、換刀時間、主軸啟停時間等。文件也指出，沒有準確的機台特徵參數，即使演算法完整，仍然無法產生可靠預測。

GUI 則提供使用者操作：選擇 NC 檔、選擇機台 Profile、執行估測、查看結果、匯出報告。

## 核心資料結構設計

可以先用 Python dataclass 或 Pydantic model。

```python
from dataclasses import dataclass
from typing import Optional, Literal

@dataclass
class ModalState:
    motion: Optional[str] = None      # G00/G01/G02/G03
    plane: str = "G17"               # G17/G18/G19
    distance_mode: str = "G90"       # G90/G91
    unit: str = "mm"                 # mm/inch
    feed_mode: str = "G94"           # G94/G95/G93
    feedrate: Optional[float] = None
    spindle_speed: Optional[float] = None
    coolant_on: bool = False

@dataclass
class MotionBlock:
    line_no: int
    raw: str
    motion_type: Literal["rapid", "linear", "arc_cw", "arc_ccw", "dwell", "tool_change", "mcode"]
    start: tuple[float, float, float]
    end: tuple[float, float, float]
    feedrate: Optional[float]
    length: float = 0.0
    estimated_time: float = 0.0
    warning: Optional[str] = None
```

## GUI 開發建議

若要做給內部工程師或現場人員使用，建議採用 PySide6。

GUI 主畫面可分成五個頁籤。

第一個頁籤是「檔案與專案」：

```
- 選擇 NC-Code
- 顯示程式名稱
- 顯示行數
- 顯示刀具數量
- 顯示解析狀態
```

第二個頁籤是「機台設定」：

```
- 選擇機台 Profile
- 編輯快速進給速度
- 編輯最大切削進給
- 編輯加速度
- 編輯 Jerk
- 編輯換刀時間
- 儲存 Profile
```

第三個頁籤是「估測結果」：

```
- 總加工時間
- 快速移動時間
- 切削時間
- 暫停時間
- 換刀時間
- 輔助時間
- 估測可信度
```

第四個頁籤是「分段明細」：

```
- line no
- NC-Code
- motion type
- start position
- end position
- length
- feedrate
- time
- warning
```

第五個頁籤是「視覺化」：

```

- 第二階段增加速度/加速度/Jerk 曲線
```
